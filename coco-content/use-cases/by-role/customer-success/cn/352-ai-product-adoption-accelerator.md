# #352 — AI 产品采用加速助手

**角色**: 客户成功
**行业**: SaaS / 科技
**任务**: 自动化
**标识符**: `ai-product-adoption-accelerator`

---

## 简介

**痛点：低采用率是正在进行中的无声流失——CS 团队发现时往往已太迟**

产品采用是 SaaS 客户留存平衡的支点。深度使用产品的客户——激活多项功能、将其融入核心工作流、围绕其能力建立习惯——在管理良好的 SaaS 业务中续约率高达 90% 以上。停留在表面的客户——只使用一两个功能、进行孤立的实验而没有集成、从未将产品完全嵌入日常运营——的流失率则显著更高。SaaS 领域的研究结论一致：功能采用广度是续约最强的预测指标之一，而且相关性是因果性的，而非仅仅是相关性。

CS 团队面临的运营问题是监控缺口。表明低采用率的使用数据存在——在产品分析平台中——但 CS 团队很少对账户级别的数据有系统性可见度，即使有，将使用数据转化为主动外联也是额外的手动步骤，大多数 CSM 不会持续执行。典型模式是 CSM 在续约对话开始、季度业务回顾揭示疲弱的参与指标，或客户要求降级或取消时才意识到低采用率。到那时，客户已经形成产品没有提供价值的固定看法，干预必须同时克服使用缺口和负面认知——这比从一开始就防止使用缺口形成要难得多。

个性化问题使监控缺口更加复杂。一个账户的低采用率可能源于与另一个具有相同使用指标的账户完全不同的原因。一个客户可能经历了高关键联系人更换，失去了推动采用的内部倡导者。另一个可能经历了从未扩展到第一个团队之外的成功初始部署。第三个可能有需要从未完成的特定集成的工作流。第四个可能从未接受过关于使产品对其用例最有价值的功能的培训。通用的"更多使用产品"外联之所以无效，恰恰是因为它忽视了这些不同的因果模式。

规模挑战也是真实的。管理 50 个账户的 CSM 无法在不大幅增加每账户时间投入的情况下，为每个低采用客户制作真正个性化的、特定于用例的赋能计划。在标准覆盖比率下，数学不成立，这就是为什么通用数字外联是默认做法，即使 CSM 知道它表现不佳。COCO 通过使以标准人工努力无法维持的规模生成真正个性化的赋能计划和外联变得可行，改变了这个方程式。

**COCO 如何解决这一问题**

COCO 使 CS 团队能够大规模监控采用情况、识别每个账户中的具体采用缺口、生成个性化赋能计划，并产生针对阻止更深入产品使用的实际障碍的有针对性外联。

1. **采用缺口识别和细分**：COCO 分析产品使用数据，识别每个客户正在和未在使用哪些功能和能力——构建区分由不同结构因素导致的低采用的采用档案。
   - 按客户细分生成采用热图，显示组合中的功能使用情况
   - 按可能的原因类型对低采用账户进行细分，以便差异化响应

2. **特定用例赋能计划生成**：COCO 为未充分利用的客户生成个性化赋能计划——根据其用例识别哪些具体功能最能改善其成果，并以合理的进展序列规划采用旅程。
   - 创建与客户声明的业务目标挂钩的账户特定采用路线图
   - 按每个客户具体情况的预期价值影响优先排序功能推荐

3. **个性化外联起草**：COCO 为每个低采用账户起草有针对性的外联信息——将特定的未使用功能与客户已知的用例和业务背景相连接，而非发送通用的产品更新通知。
   - 按账户健康层级和采用缺口类型生成外联变体
   - 创建带有基于响应的升级逻辑的外联序列（初始 + 2 次跟进）

4. **按细分设计采用活动**：COCO 帮助 CS 领导者为特定客户细分设计系统化采用活动——构建具有相似采用档案的多个账户协调活动的信息、内容和外联序列。
   - 设计带有内容和外联组件的多触达点采用活动
   - 生成按细分衡量采用提升的活动报告框架

5. **培训和资源推荐**：COCO 识别与每个客户采用缺口最相关的具体培训资源、帮助文档和客户教育内容——生成 CSM 可以在赋能对话中分享的个性化资源指南。
   - 从现有文档和培训内容中为每个采用缺口整理相关资源
   - 起草说明每个资源为何相关的个性化资源共享电子邮件

6. **采用进度跟踪和报告**：COCO 为个别账户和客户组合生成采用进度报告——跟踪相对于基线的改善、标记干预产生采用提升的账户，以及识别需要额外干预的账户。
   - 生成供 CSM 审阅的账户级采用趋势报告
   - 为 CS 领导层生成客户组合采用健康摘要

**可量化的成果**

- **采用干预时机**：使用 COCO 进行采用监控的 CS 团队比依靠续约周期发现的团队平均提前 70 天识别低采用账户
- **功能激活率**：COCO 生成的个性化赋能计划与通用外联活动相比产生 2.3 倍更高的功能激活率
- **外联响应率**：有针对性的、特定于用例的采用外联实现 35-45% 的响应率，vs. 通用产品更新邮件的 8-12%
- **采用改善带来的续约提升**：在续约前 6 个月功能采用广度增加 2 个以上功能的账户，与对照账户相比续约率高 28%
- **CSM 时间效率**：COCO 辅助的采用活动设计和外联起草将每账户赋能工作量减少 60%，使 CSM 能够同时在 3 倍以上的账户上运行采用干预

**受益角色**

- **拥有中到大型账户组合的客户成功经理**：需要大规模进行主动采用干预，同时不牺牲使这些干预有效的个性化
- **CS 运营和数字 CS 团队**：大规模设计和运行自动化采用计划，需要 AI 辅助个性化来提高传统模板表现不佳的数字外联的绩效
- **产品团队**：希望了解哪些功能在哪些客户细分中采用缺口最大，以便优先考虑具有最高留存影响的产品内指导改进和入职流程变更
- **客户成功副总裁**：对净收入留存负责，需要系统性证据表明其团队正在主动推动采用，而非等待续约信号触发补救措施

## 实用提示词

**提示词 1 — 客户细分的采用缺口分析**
```
我有我的 [企业] 客户细分的产品使用数据，想识别代表最高流失风险的采用缺口。我管理 [X] 个企业账户，有以下数据：

功能使用数据（对每个账户，关键功能激活了多少百分比）：
[粘贴使用数据摘要或描述：例如，"功能 A：85% 的账户激活；功能 B：62%；功能 C：23%；功能 D：31%；功能 E：67%"]

产品背景：[描述产品的关键功能以及它们使客户能够做什么]

请：
1. 识别整个细分中采用率最低的功能
2. 对于每个低采用功能，评估：采用率低的可能原因是什么？（复杂性、发现困难、价值主张不清晰、培训缺口、集成要求）
3. 识别采用缺口最严重的账户（相对于其合同层级使用功能最少）
4. 按采用风险优先级对账户进行排名——哪些应立即收到 CSM 外联？
5. 建议这是否最好通过以下方式解决：（a）个别 CSM 外联，（b）细分级数字活动，或（c）产品侧改进
```

**提示词 2 — 个性化采用赋能计划**
```
我有一个严重未充分利用我们产品的客户，我需要为他们设计个性化的采用赋能计划。请帮我创建一个账户特定的计划。

客户背景：
- 公司：[名称、规模、行业]
- 合同：[层级/套餐、价值、合同开始日期]
- 主要用例：[他们购买的原因，他们正在解决的业务问题]
- 当前使用：[他们使用哪些功能，哪些不用]
- 计划中包含但他们未使用的功能：[列表]
- 我了解的他们的业务目标：[从季度业务回顾、电话、对话中]
- 之前的赋能尝试：[此前尝试过什么，结果如何]

请设计 90 天采用赋能计划，包括：
1. 根本原因假设：为什么这个客户不使用 [特定功能]？
2. 功能采用优先路线图：他们应该首先采用哪 2-3 个功能，以什么顺序，为什么？
3. 逐周参与计划：接下来 12 周每周应该发生什么？
4. 资源包：哪些培训、文档或示例最能支持每个功能采用步骤？
5. 成功里程碑定义：对于这个客户，30、60 和 90 天时"良好采用"是什么样子？
```

**提示词 3 — 低采用外联邮件序列**
```
我需要联系我的客户组合中功能采用明显低于同层级可比账户的 [X] 个账户。我希望外联真正有帮助和针对性——不是通用的"检查"或产品更新邮件。

账户细分背景：
- 行业：[行业]
- 公司规模：[典型规模范围]
- 他们的主要用例：[描述]
- 他们未充分利用的功能：[列出这些账户大多数缺少的 2-3 个功能]
- 他们正在使用的功能（锚点）：[他们确实使用什么——这是"他们已经在使用 X"的钩子]
- 那些未使用功能将实现的业务成果：[如果他们使用这些功能，他们会实现什么？]

请写一个 3 封邮件的外联序列：
邮件 1（第 1 周）：引用其特定用例的开场，介绍一个具体的未使用功能，带有清晰的"这对你意味着什么"框架，以及清晰、低摩擦的行动号召
邮件 2（第 2 周，如无响应）：不同角度——分享关于类似公司如何使用此功能的客户故事或数据点，带有略微不同的行动号召
邮件 3（第 4 周，如无响应）：直接询问——"花 20 分钟了解 [功能] 会有帮助吗？"带有具体会议链接

每封邮件保持在 200 字以内。使用温暖、顾问式的语气——不是产品推销。
```

**提示词 4 — 客户细分的采用活动设计**
```
我想为我的客户组合中所有共享类似采用缺口的 [X] 个账户运行协调的采用活动：他们都没有使用 [功能名称/功能类别]。这些账户都属于 [描述细分：同一层级、同一行业或同一用例]。

功能背景：[描述功能、它做什么以及它为使用它的客户提供什么业务价值]
活动中的账户：[X 个账户，总年经常性收入：$X]
活动目标：[您的目标是在 90 天内实现什么百分比的功能激活率？]
可用资源：[您有哪些培训内容、演示材料或文档可以分享]

请设计 90 天采用活动，包含：
1. 活动信息框架：什么是使此功能对该细分特定用例具有吸引力的核心价值信息？
2. 多渠道外联序列：邮件节奏、应用内消息建议以及 CSM 通话时机
3. 内容日历：何时分享什么，以什么格式（演示视频、书面指南、客户示例、现场网络研讨会）
4. 层级化：高价值账户是否应该在此活动中受到与低价值账户不同的对待？
5. 成功指标和跟踪：我们将如何在 30、60 和 90 天衡量活动有效性？
6. 起草我将发送给所有 [X] 个账户以启动活动的活动开幕邮件
```

```
我需要为我的 CS 总监准备月度采用健康报告。请帮我根据我的客户组合数据构建和叙述这份报告。

客户组合概览：
- 总账户数：[X]
- 高采用账户（4 个以上核心功能激活）：[X — X%]
- 中等采用账户（2-3 个核心功能）：[X — X%]
- 低采用账户（少于 2 个核心功能）：[X — X%]
- 与上月相比采用情况改善的账户：[X]
- 与上月相比采用情况下降的账户：[X]

需要立即关注的账户（带关键数据列表）：
[对每个被标记的账户：账户名称、层级、年经常性收入、采用分数、关键缺口、续约日期]

进行中的干预：
[描述本月运行的任何活跃采用活动或个别外联序列]

请生成：
1. 为我的 CS 总监准备的 200 字客户组合采用健康叙事摘要——数字意味着什么、什么在改善、什么令人担忧
2. 重点解决最高优先级采用缺口的未来 30 天建议行动计划
3. 识别任何系统性模式——是否有多个账户在同一特定功能上挣扎？
4. 基于此数据为我与 CS 总监下次一对一会议推荐的对话主题
```
