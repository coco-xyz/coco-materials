# 注意事项

使用 COCO AI 员工时的重要提示和常见机制说明。

## 群聊与私聊白名单

### 私聊（1对1）

AI 员工默认支持私聊。在 Telegram 或 Lark/飞书 中直接搜索你的 Bot 名称，发起私聊对话即可使用，无需额外配置。

### 群聊

在群组中使用 AI 员工时，需要注意以下几点：

| 项目 | 说明 |
|------|------|
| 触发方式 | 必须 **@你的Bot** 才会触发响应，普通群消息不会触发 |
| 群权限 | Bot 需要被添加到群组中才能响应（参见[渠道部署指南](./channel-deployment)的第7步） |
| 飞书权限 | 必须开通 `im:message.group_at_msg:readonly` 权限，否则 Bot 无法接收群内 @消息 |
| 群成员范围 | Bot 被添加到群后，该群所有成员都可以 @Bot 对话 |

> **提示：** 如果 Bot 在群里不响应，请检查：
> 1. Bot 是否已添加到该群组
> 2. 是否使用了 @提及（而非普通消息）
> 3. 飞书应用是否已开通 `im:message.group_at_msg:readonly` 权限

### 多群组使用

一个 AI 员工可以同时被添加到多个群组。每个群组的对话上下文是独立的，不会相互干扰。

## 长期记忆与 SOP 存储

AI 员工具备长期记忆能力，可以记住重要的工作流程（SOP）、偏好设置和业务规则，在后续对话中持续遵循。

### 如何存储 SOP

直接在对话中告诉 AI 员工你希望它记住的内容：

**示例：**

```
请记住以下工作流程：
1. 每天早上9点检查邮箱中的客户询盘
2. 将询盘内容整理成标准格式：客户名称、需求描述、联系方式
3. 按紧急程度分为A/B/C三级
4. A级当天回复，B级48小时内回复，C级一周内回复
```

```
请记住：我们公司的品牌色是 #F5C542，所有设计稿必须使用这个颜色。
```

```
请记住：回复客户邮件时，结尾统一使用以下签名：
Best regards,
[姓名]
[公司名] | [职位]
```

### 记忆管理

| 操作 | 说明 |
|------|------|
| 添加记忆 | 对话中说「请记住...」或「以后都要...」 |
| 查看记忆 | 问「你记住了哪些规则？」或「列出我之前告诉你的所有SOP」 |
| 修改记忆 | 说「把之前的XX规则改为...」 |
| 删除记忆 | 说「忘掉之前关于XX的规则」 |

> **提示：** 长期记忆在同一个 AI 员工实例内持续有效，即使重新开始对话也会保留。建议将重要的业务规则、工作流程、格式要求等存入长期记忆。

### 适合存入长期记忆的内容

- **工作流程（SOP）**：日常操作步骤、审批流程、汇报格式
- **品牌规范**：品牌色、字体、tone of voice、禁用词
- **业务规则**：定价策略、折扣权限、服务条款
- **沟通偏好**：邮件格式、回复模板、语言风格
- **团队信息**：组织架构、对接人、职责分工

## 其他常见注意事项

### 消息长度限制

| 渠道 | 单条消息限制 | 超长消息处理 |
|------|------------|------------|
| Telegram | 4096 字符 | 自动拆分为多条消息发送 |
| Lark/飞书 | 约 30,000 字符 | 超长内容会自动截断 |

### 文件处理

AI 员工支持处理以下类型的文件（需开通 `im:resource` 权限）：

- **文档**：PDF、Word、Excel、PPT
- **图片**：PNG、JPG、GIF
- **文本**：TXT、CSV、JSON、Markdown

> **注意：** 文件大小建议不超过 20MB。大文件处理可能需要较长时间。

### 响应时间

| 场景 | 预计响应时间 |
|------|------------|
| 简单问答 | 3-10 秒 |
| 文件处理 | 10-30 秒 |
| 复杂分析 | 30-60 秒 |
| 长文档生成 | 1-3 分钟 |

> **提示：** 如果 AI 员工长时间未响应（超过 2 分钟），可以尝试重新发送消息。

### 安全与隐私

- AI 员工不会将你的对话数据分享给其他用户
- 每个 AI 员工实例的数据是隔离的
- 建议不要在对话中发送敏感的密码、密钥等信息
- 企业版支持自定义数据保留策略
