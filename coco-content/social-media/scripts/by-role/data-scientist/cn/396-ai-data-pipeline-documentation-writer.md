**场景 1（0:00–0:10）：钩子**
[画面]：新数据工程师接手一条关键 Airflow 流水线，没有文档，Slack 消息写着"原作者上个月离职了"

**场景 2（0:10–0:30）：问题**
[画面]：新工程师花 4 天阅读 DAG 代码，试图理解一个 CASE WHEN 语句背后的业务逻辑；凌晨 3 点的告警，没有运行手册，错误的修复方案应用了，下游模型被污染；合规审计员要求提供流水线文档，收到的是一个 GitHub 链接

---

**场景 3（0:30–0:50）：COCO 实操**
[画面]：将 Airflow DAG 和 dbt SQL 连同业务背景说明一起粘贴给 COCO；COCO 生成：流水线概述、Mermaid 数据流图、带业务逻辑的分步描述、假设表、故障模式，以及带分步事故处理程序的运行手册
[旁白]："将流水线代码粘贴给 COCO，加几句业务背景说明。COCO 生成完整文档：数据血缘、业务逻辑解释、假设条件、已知边缘情况，以及包含分步事故处理程序的操作运行手册。"

---

**场景 4（0:50–1:00）：结果**
