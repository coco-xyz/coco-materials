# #347 — AI 客户成功手册编写器

**角色**：客户成功
**行业**：SaaS / 科技
**任务**：文档
**标识符**：`ai-customer-success-playbook-writer`

---

## 简介

每个高绩效的客户成功团队都依据一套共享手册运作：这些文档化协议告诉CSM在客户遇到特定情况时应该做什么——无论是QBR、续约对话、风险升级还是执行层赞助人变更。这些手册是CS组织的智识资本，以可复用的格式编码了团队的最佳实践、来之不易的经验教训和战略框架，无论经验水平如何，都能提升每位CSM的基准能力。

问题在于大多数CS团队没有将这些手册书面化——或者即便有，文档也不完整、过时，或以"我们一直是这么做的"的形式存在于某人的脑海中。当新CSM加入时，他们要经历90天的适应期：跟随资深同事、通过观察吸收隐性知识，并缓慢形成自己版本的手册——而这个版本未必与团队认为的最佳实践相符。这是昂贵的：90天的适应期意味着90天次优的客户覆盖，对于有积极增长目标的公司而言，意味着前三个月的客户总是比由资深CSM管理的客户获得更差的体验。

第二个问题是手册债务。即使拥有手册的团队也不更新它们。18个月前编写的手册没有考虑到产品变化、新的竞争格局、不断演变的客户期望，或从最近20次流失事后分析中吸取的教训。陈旧的手册给CSM虚假的信心：他们遵循记录的流程，得到糟糕的结果，然后不知道是该责怪手册还是自己的执行。

COCO解决了这两个问题。它通过结合团队的机构知识（以笔记、通话录音摘要或口头描述的形式表达）与CS最佳实践，帮助CS团队从零生成全面、结构化的手册——然后通过不断融入新教训，帮助团队定期更新这些手册。

**COCO如何解决这一问题——分步说明：**

1. **情境定义**：CS负责人或CSM定义手册应处理的具体情境（如"一级账户的执行层赞助人变更"、"30天未登录的客户"、"与贬损者的续约对话"）。

2. **手册结构生成**：COCO生成完整的手册结构——情境触发点、立即行动、干系人映射、沟通模板、升级路径、成功标准和常见失败模式。

3. **行动序列起草**：COCO将手册中的每个步骤写成具体、可执行的指令——不是模糊的指导原则，而是具体的"做这个，然后做这个，使用这个模板"。

4. **模板和脚本嵌入**：COCO生成直接插入手册步骤的嵌入式模板（邮件、通话脚本、内部升级消息）。

5. **失败模式记录**：COCO识别该手册通常无法产生期望结果的前3种方式，并为每种方式包含缓解指令。

6. **审查和更新协议**：COCO生成建议的审查节奏和应触发手册修订的触发事件集（如"每次此细分的流失后审查"）。

**可量化成果：**

- 新CSM达到完全生产力的适应时间：从11周 → 6周
- 手册覆盖率（已记录情境/已知情境总数）：从28% → 91%
- 跨经验水平的CSM结果一致性：初级/资深绩效差距减少54%
- 创建一份全面新手册的时间：从12+小时 → 90分钟
- 有记录vs无记录手册细分的流失率：9.1% vs 18.3%

**受益角色：**

- **新CSM**：通过清晰、可执行的手册更快上手，而无需依赖部落知识和跟随观察。
- **有经验的CSM**：减少回答"遇到...我应该怎么做"问题的时间，更多时间用于高杠杆工作。
- **客户成功副总裁**：建立一个可扩展、可审计的CS运营，质量不取决于员工资历——并可以自信地招聘以实现规模增长。
- **客户**：无论分配的是两年还是两个月经验的CSM，都能体验到一致的高质量参与。

## 5个实用提示词

**提示词1 — 完整手册生成**
```
为以下情境编写一份全面的客户成功手册：

情境：[情境——如"一级企业账户的执行层赞助人离职或被替换"]
适用客户细分：[细分——如企业客户，ARR $5万+]
典型时间线：[CSM有多少时间采取行动]
处理不当的主要风险：[风险]
CSM可用资源：[资源——如高管升级通道、礼品预算、法律联系人]

手册必须包含：
1. 触发条件定义（手册确切的激活时机）
2. 立即行动（前48小时）
3. 第1周行动
4. 第2-4周行动
5. 关键干系人沟通模板（最少3个）
6. 升级路径和标准
7. 成功定义（60天时"处理好"的样子是什么？）
8. 前3个失败模式及如何避免
```

**提示词2 — 续约手册**
```
为续约价值$[范围]的[客户细分]账户创建90天续约手册。包含：

- 第90天：触发和内部审查
- 第60天：第一次高管外联
- 第45天：正式续约对话
- 第30天：提案和谈判阶段
- 第14天：最终成交活动
- 第0天及之后：续约确认和下一年目标设定

对每个阶段，包含：CSM行动、干系人沟通（附模板）、常见障碍和升级标准。同时包含：如何处理价格异议、如何处理竞争对手评估，以及如何处理不回应的内部倡导者。
```

**提示词3 — 手册更新和差距分析**
```
审查以下[情境]的现有CS手册，并：
1. 识别过时或不再符合当前产品功能的步骤
2. 基于常见失败模式识别缺失的步骤
3. 识别应该更新的语言或模板
4. 基于当前CS最佳实践建议3项改进

当前手册：
[粘贴现有手册]

近期经验教训（来自流失事后分析或团队回顾）：
[粘贴教训]

过去6个月与此手册相关的产品变化：
[列表]
```

**提示词4 — 快速参考手册卡片**
```
将以下全面CS手册转换为CSM可在通话或会议中实时使用的单页快速参考卡片。只包含：触发条件、前3个行动、关键谈话要点，以及最重要的一件不该做的事。

完整手册：
[粘贴手册]

格式化为无需滚动即可在一个屏幕上显示的结构化卡片。
```

**提示词5 — 手册库索引**
```
我们需要建立一个全面的CS手册库。根据以下账户投资组合档案和历史流失数据，识别我们应该优先建立的15个手册。

投资组合档案：
- 总账户数：[N]
- 细分：[列表]
- 主要服务行业：[列表]
- 常见流失原因（来自事后分析）：[列表]
- 当前已记录手册：[列表]

对每个推荐手册：
- 手册名称和涵盖的情境
- 优先级（P1/P2/P3）
- 使用COCO构建的估计时间
- 如果实施的预期对留存率的影响
```
