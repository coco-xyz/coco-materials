# #323 — AI 职位描述撰写助手

**角色**: 人力资源
**行业**: 企业
**任务**: 内容创作
**标识符**: `ai-job-description-writer`

---

## 简介

撰写一份高质量的职位描述，是人力资源工作中最容易被低估的挑战之一。表面上看，只需列出职责、要求和福利待遇，然后发布招聘信息即可。但实际上，HR 专业人员和用人部门负责人往往需要在 HR、招聘经理和法务团队之间来回沟通，花费大量时间才能产出一份同时满足吸引顶尖人才、准确反映岗位职责、符合法律合规要求以及契合公司雇主品牌的职位描述。

行业研究显示，当招聘流程涉及多方评审时，企业平均每份职位描述需耗费 3 至 5 小时。对于每年发布 50、100 甚至 500 个职位的规模化招聘企业而言，这意味着每年数百小时的生产力浪费。更严重的是，低质量的职位描述会引发一系列连锁问题：吸引到不匹配的候选人、拉长招聘周期、因岗位描述与实际工作不符导致新员工早期离职，以及因措辞不当对受保护群体构成歧视而产生法律风险。

LinkedIn 2023 年人才趋势报告指出，72% 的招聘经理认为职位描述的质量直接影响候选人管道的质量。然而，大多数企业缺乏标准化模板，往往沿用过时的旧版描述，或完全依赖工作繁忙、缺乏 HR 与文案专业知识的用人部门经理来撰写。结果产出的是内容不一致、术语堆砌或措辞模糊的招聘信息，无法在竞争激烈的人才市场中有效塑造雇主差异化形象。

COCO 通过融合 HR 最佳实践、行业基准和雇主品牌原则，成为 HR 专业人员的智能写作伙伴，从根本上改变了这一工作流程。HR 人员无需从空白页面开始，只需向 COCO 提供一份结构化的简报——岗位名称、所属团队、核心职责、必备和优先资质、汇报关系及相关背景——COCO 即可在数秒内生成一份完整、精炼的职位描述。

**COCO 的解决步骤：**

1. **需求收集**：HR 或招聘经理向 COCO 提供简短的结构化提示，包括岗位名称、部门、级别、3-5 项核心职责、必备资质及团队文化特色，无需特殊格式。

2. **草稿生成**：COCO 自动生成包含所有标准模块的完整职位描述：岗位概述、工作内容、任职要求、加分项、团队介绍及薪酬区间建议，全程使用主动语态，避免冗余术语，并根据岗位级别进行语气校准。

3. **品牌语调适配**：HR 团队可指定 COCO 匹配特定的雇主品牌风格——正式商务、创业轻松或使命驱动——COCO 将相应调整词汇选择、句式结构和内容侧重，确保全公司发布内容风格统一。

4. **合规审查**：COCO 自动标记潜在问题措辞——涉及年龄偏见（如"年轻有活力"）、性别化语言或不必要的体能要求——在职位描述定稿前降低法务审核轮次。

5. **迭代优化**：如果招聘经理希望加强对成长机会的描述，或法务要求添加特定免责声明，HR 只需向 COCO 发出跟进指令，即可立即获得修订版本。

6. **多格式导出**：COCO 可将同一职位描述重新格式化，适配不同发布平台——更长的 LinkedIn 版本、针对招聘网站优化的简短版本，或内部职位公告——无需 HR 团队重新撰写。

**可量化成果：**

- 职位描述撰写时间从 3-5 小时缩短至 30 分钟以内，效率提升 85%
- 全公司职位发布内容一致性评分提升 60%
- 由于初稿质量更高，法务审核轮次从平均 2.3 轮降至不足 1 轮
- 试点企业候选人质量评分（招聘经理对候选人与岗位匹配度的评价）提升 34%
- 因职位描述准确反映岗位要求，平均招聘周期缩短 8 天

**受益角色：**

- **HR 业务伙伴（HRBP）**：全程负责招聘流程，需要在多个开放职位中快速产出高质量内容
- **人才招募专家**：负责维护发布内容一致性、代表雇主品牌形象、管理候选人管道质量
- **用人部门经理**：对岗位有深入了解，但缺乏文案专业技能或时间，需要 HR 框架支撑
- **HR 运营/合规团队**：需确保所有职位描述在发布前符合监管标准

## 实用提示词

**提示词 1 — 从零开始生成完整职位描述**
```
请为 [一家拥有 300 名员工的 B2B SaaS 公司] 撰写一份 [高级产品经理] 的职位描述。该职位隶属于 [平台] 团队，向 [产品副总裁] 汇报。核心职责包括 [路线图规划、与工程团队的跨部门协作以及客户需求调研]。任职要求：[5 年以上产品管理经验、具备 API 产品经验、较强的数据分析能力]。加分项：[金融科技领域经验、MBA 学位]。我们的文化氛围 [协作包容、低自我主义]。请使用积极、直接的语气，避免企业套话。
```

**提示词 2 — 改写现有职位描述**
```
以下是我们现有的 [数据分析师] 职位描述：[粘贴现有文本]。请按以下要求进行改写：(1) 以岗位影响力开篇，而非泛泛的公司介绍；(2) 全文使用主动语态；(3) 将任职要求拆分为"必备条件"和"优先条件"两个模块；(4) 添加简短的"为什么选择我们"板块，突出 [远程优先文化和丰厚的学习预算]。
```

**提示词 3 — 同一职系多级别职位描述**
```
我们正在招聘同一职系的三个级别：[初级、中级和高级软件工程师]。请撰写三份职位描述，共享一个关于 [公司名称] 工程文化的通用开篇，同时清晰区分每个级别的经验要求、职责范围和发展机会。请保持三份文档格式统一。
```

**提示词 4 — 雇主品牌风格适配**
```
请将这份标准的 [HR 通才] 职位描述，改写成符合我们雇主品牌调性的版本。我们的品牌核心价值是：[大胆、人性化、以成长为驱动]。常用品牌语句包括："我们创造有意义的事物"、"没有官僚主义"、"你主导自己的职业发展"。删除被动语态，将"有竞争力的薪酬"等模糊表述替换为具体信息：[薪酬区间 65,000–80,000 美元，20 天带薪假，每年 2,000 美元学习补贴]。
```

**提示词 5 — 合规与偏见审查**
```
请审查以下 [运营经理] 职位描述，识别：(1) 可能无意中排除受保护群体的措辞（性别化词汇、年龄偏见、不必要的体能要求）；(2) 可用具体标准替代的模糊限定词；(3) 缺失的法律建议条款（平等就业机会声明、合理便利通知）。请提供修正后的完整版本。原始职位描述：[粘贴文本]。
```
