# Use Case #089: AI社交媒体监听助手

**Role**: Marketing | **Industry**: E-commerce, SaaS | **Task**: Social Listening, Brand Monitoring

---
## 详细介绍

**痛点：互联网正在谈论你而你毫不知情**

你的品牌每年在社交媒体、论坛、评价网站、新闻媒体和博客上被提及250万次。你监控到的大约只有5%。其余95%——包括即将因客户投诉而病毒式传播的推文、竞争对手在Reddit上偷走你叙事的讨论帖、以及刚刚自发赞扬你产品的意见领袖——对你来说都是不可见的。

在线对话的规模已经超出人工监控能力好几个数量级。仅Twitter每天就有5亿条帖子。Instagram、TikTok、LinkedIn、Reddit、Quora、YouTube评论、应用商店评价、行业论坛、Hacker News——品牌相关对话发生的场景正在以任何团队都无法追踪的速度碎片化。

这种盲区的后果很严重。96%的不满客户从不直接向你投诉——他们向其他所有人投诉。当客户服务问题通过传统渠道浮出水面时，它已经在社交媒体上被数百甚至数千人看到了。社交平台上对响应时间的期望现在不超过一小时，但品牌平均响应时间为5-12小时。每延迟一小时，客户满意度下降15%。

舆情追踪同样失灵。营销团队依赖季度品牌感知调查来捕捉某一时刻的快照。但品牌情绪每天都在变化——一条病毒式帖子可能在一夜之间扭转局面。等季度结果出来时，损害已经造成或机会已经错过。你在看着后视镜开车。

危机检测是差距最危险的地方。社交媒体危机呈指数级升级：投诉变成讨论帖，变成话题标签，变成新闻报道。在第一个小时发现危机的公司可以控制局面。6小时后才响应的公司面临10倍的声誉损害和恢复成本。人工监控根本无法提供所需的速度。

竞争情报也是受害者。竞争对手的产品发布、定价变化、客户投诉和战略信息都在社交媒体上公开上演。但没有系统性监控，这些信号就淹没在噪声中。

**COCO如何解决**

COCO的AI社交媒体监听助手作为7×24小时品牌情报系统运作，覆盖所有相关平台：

1. **多平台监控**：COCO持续扫描Twitter/X、Instagram、LinkedIn、Reddit、TikTok、YouTube、新闻网站、博客、评价平台（G2、Trustpilot、App Store）和行业论坛。实时监控品牌提及、产品名称、竞品名称、行业关键词和高管提及。

2. **情感分类**：每条提及都经过带有上下文理解的情感分析（正面、负面、中性）。COCO区分讽刺和真实赞扬，识别投诉背后的情绪（沮丧vs.失望vs.愤怒），并追踪具有统计显著性的情感趋势。

3. **趋势检测**：COCO在话题达到高峰前识别新兴主题和对话。它追踪提及速度——对话量增长率——以发现发展中的趋势。当与你品牌相关的话题出现异常加速时，你在几分钟内就能知道，而不是几天后。

4. **危机预警**：当负面提及超过基线阈值3倍或更多时，COCO立即触发危机警报，附带严重性评估、原始来源、当前传播速率、建议响应策略和供快速审批的草拟回应。这通常比人工检测提前6小时以上预警。

5. **响应起草**：对于需要回应的提及——客户投诉、产品问题、错误信息——COCO起草符合品牌语调的上下文适当回应。回应排队等待人工审核和一键批准，将响应时间从数小时缩短至数分钟。

6. **意见领袖识别**：COCO识别在品牌对话中具有超大影响力的个人——包括积极倡导者和潜在贬低者。按覆盖面、互动率、受众相关性和情感轨迹对意见领袖评分，实现有针对性的关系建设。

**可衡量的成果**

- **97%的提及覆盖率**（从5%提升），确保几乎没有品牌相关对话被遗漏
- **响应时间从12小时缩短至18分钟**，满足现代消费者对社交互动的期望
- **正面品牌情感提升3.4倍**，得益于主动互动和更快的问题解决
- **危机检测提前6小时**，大幅减少声誉损害
- **社交互动率提升156%**，通过对自然对话的及时、相关回应实现

**谁能受益**

- **营销团队**：实时品牌情报看板，提供可执行的洞察而不仅是数据堆砌
- **公关传播**：早期危机预警和可快速部署的草拟回应
- **客户支持**：社交提及自动分类路由，附带草拟回应
- **产品团队**：按主题聚合的未过滤客户反馈，从自然对话中浮现的功能需求

## 实用提示词

**提示词 1: 全面品牌提及分析**
```
分析我们品牌在过去[时间段]的社交媒体提及：

品牌名称：[名称]
同时监控：[产品名、常见拼写错误、话题标签、高管姓名]
覆盖平台：Twitter/X、LinkedIn、Reddit、Instagram、TikTok、YouTube、G2、Trustpilot、Hacker News

分析内容提供：
1. 数量概览：每个平台的总提及量、每日趋势线、与上一期间对比
2. 情感分布：每个平台的正面/负面/中性百分比，各附示例
3. 热门主题：品牌提及中最常见的10个话题，每个附数量和情感倾向
4. 值得关注的提及：来自1万+粉丝账号的提及、媒体报道、或病毒式内容（50+互动）
5. 竞品对比：我们的声量份额与[竞品1、竞品2、竞品3]的对比
6. 客户投诉：按问题类型、频率和严重性对所有负面提及分类
7. 赞誉与倡导：识别自发品牌倡导者及他们赞扬的具体方面
8. 新兴话题：最近7天出现的、之前不存在的新主题

格式化为高管看板，顶部为关键指标，下方为详细分析，并基于发现提供5条建议行动。
```

**提示词 2: 社交媒体危机检测与响应**
```
检测到潜在危机。分析情况并准备响应计划：

触发事件：[描述引发的帖子/事件/投诉]
当前状态：[提及数量、传播速率、受影响平台]
情感倾向：[描述整体语气——愤怒、失望、嘲讽等]
关键声音：[是否有意见领袖或媒体参与]
我们目前的回应：[描述已采取的行动或"尚未回应"]

请提供：
1. 严重性评估：1-10评分并说明理由。考虑：提及速度、意见领袖参与、媒体报道可能性、声称的事实准确性、监管影响
2. 情况摘要：适合高管阅读的3句话简明摘要
3. 利益相关者影响：谁受影响（客户、合作伙伴、投资者、员工）以及如何影响
4. 响应策略：建议方式（确认、解释、道歉、纠正、或监控）
5. 草拟回应：
   - 官方声明（50-100字，适用于所有平台）
   - 社交媒体回复模板（用于个别回应）
   - 面向客户团队的内部FAQ（10个预计问题及答案）
6. 不要回应的内容：具体避免说什么以及为什么
7. 监控计划：未来24/48/72小时需要关注什么
8. 升级标准：何时升级至法务、高管层或外部公关公司

为每个行动项标注时间线、责任人和紧急级别。
```

**提示词 3: 竞品社交情报报告**
```
基于社交媒体活动生成竞品情报报告：

我们公司：[名称]
追踪竞品：[竞品1]、[竞品2]、[竞品3]
行业：[行业]
时间段：[日期]

分析并比较：
1. 声量份额：每个品牌在行业总对话中占比。时间趋势
2. 情感对比：每个品牌的净情感分数。各自正面/负面情感的驱动因素
3. 内容策略分析：每个竞品发布的内容类型、频率、互动率、表现最好的内容主题
4. 产品提及：新功能发布、产品投诉、功能需求——客户对每个竞品产品的评价
5. 定价讨论：关于定价变化、价值感知或转换行为的公开讨论
6. 人才/文化：Glassdoor/LinkedIn上的员工情感、招聘信号、文化对话
7. 营销活动检测：基于协调的消息传递模式识别竞品活跃的营销活动
8. 机会缺口：客户对竞品表达不满且我们可以解决的领域

交付物：高管摘要（1页）、每个竞品的详细分析（各2-3页）、以及我们定位的战略建议。
```

**提示词 4: 意见领袖识别与合作策略**
```
从我们的社交媒体数据中识别和评估潜在的品牌意见领袖和倡导者：

品牌：[名称]
行业/细分：[描述]
目标受众：[人口统计和兴趣]
预算范围：[如适用]

需要的分析：
1. 自发倡导者：已在无赞助的情况下正面提及我们品牌的人。按以下排名：提及频率、受众规模、互动质量、受众与我们目标人群的重叠度
2. 行业意见领袖：我们行业的头部声音但尚未提及我们，其受众与我们目标匹配。包括：粉丝数、互动率、内容风格、品牌亲和信号
3. 微型意见领袖：在我们细分领域有5K-50K粉丝且高互动的账号。通常比超大意见领袖更真实且更具性价比
4. 需关注的贬低者：对我们品牌持负面情感的有影响力账号。包括负面原因和建议策略（互动、监控或忽略）
5. 平台分布：每个意见领袖在哪个平台有最强的存在和互动

对推荐的前20位意见领袖提供：
- 个人简介和内容风格
- 受众人口统计（如可用）
- 互动指标（互动率、平均评论数、分享率）
- 品牌契合度评分（1-10）及理由
- 建议联系方式（私信、邮件、公关机构、自然互动）
- 预估合作价值/成本
```

**提示词 5: 社交监听看板配置**
```
为持续的品牌监控配置全面的社交监听看板：

品牌：[名称]
产品：[列表]
竞品：[列表]
行业关键词：[列表]
高管姓名：[列表]

设计看板包含以下模块：
1. 实时信息流：为每个监控类别配置关键词查询和布尔运算符：
   - 品牌提及（包含拼写错误、缩写、话题标签）
   - 产品提及（每个产品单独列出）
   - 竞品提及（对比性对话）
   - 行业趋势关键词
   - 危机关键词（投诉、诉讼、黑客、泄露、丑闻+品牌名）

2. 告警规则：定义基于阈值的告警：
   - 提及量激增（>3倍每小时平均）→ 即时Slack告警
   - 负面情感激增（>2倍基线）→ 邮件通知公关团队
   - 意见领袖提及（>5万粉丝）→ 告警营销负责人
   - 竞品活动检测 → 每周简报发送给策略团队

3. 自动报告：
   - 每日：热门提及、情感分数、值得关注的对话、待回应队列
   - 每周：趋势分析、竞品对比、热门内容主题
   - 每月：完整品牌健康报告、声量份额趋势、意见领袖图谱

4. 响应工作流：对需要回应的提及：
   - 自动分类：投诉、问题、赞扬、错误信息
   - 使用品牌语调指南自动草拟回应
   - 基于分类路由给相应团队成员
   - 追踪响应时间和解决情况

提供完整的查询语法、告警配置和工作流自动化规则。
```
