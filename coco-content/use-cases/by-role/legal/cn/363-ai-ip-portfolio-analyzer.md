## 简介

对于科技公司而言，知识产权往往是资产负债表上最有价值的资产——然而它也是管理最不系统化的资产。运营5至10年的SaaS公司可能已积累了专利（已授权和申请中）、商标（已注册和普通法）、源代码和营销材料中的版权、算法和客户数据中的商业秘密，以及开源义务——这些分散在不同系统中，或者在许多情况下，保存在早已离职人员维护的电子表格中。

知识产权组合管理不善的后果严重且即时。因未缴年费而失效的专利永久丧失。未能抵御侵权者的商标可能因废弃而失去可执行性。未履行的开源许可义务使公司面临重大诉讼风险——多家公司因GPL执法行动而面临产品禁令和六位数和解金。并购交易常常揭示出使交易价值降低10%至30%或彻底终止交易的知识产权缺陷：被认为涵盖核心产品功能的专利实际上并不覆盖；关键源代码存储库包含GPL许可代码但没有适当的许可追踪；商标注册在错误的实体名下。

挑战不仅在于追踪，更在于战略优先级。拥有150项专利资产的公司无法以同等力度维护所有专利。关于维护哪些专利、放弃哪些、许可哪些、主张哪些以及出售哪些的决策，需要分析权利要求范围、竞争相关性、引用强度、许可收入潜力和诉讼风险——这些分析对于有经验的知识产权律师来说，每项资产需要许多小时才能进行任何深度分析。

COCO将知识产权组合管理从被动、碎片化的流程转变为主动、情报驱动的工作流：

1. **组合摄入**：知识产权团队上传组合数据——专利号、状态、年费截止日期、商标注册、版权注册、开源组件清单、许可协议。
2. **风险标记**：COCO识别即时风险——接近年费截止日期的专利、在公司有业务但未提交申请的司法管辖区的商标注册、可能未被履行的开源组件许可义务。
3. **权利要求范围分析**：对于专利资产，COCO分析权利要求语言以评估保护范围，并识别关键产品或功能是否似乎在每项专利的范围内。
4. **竞争对手映射**：给定竞争对手产品或公开已知的竞争对手专利清单，COCO评估潜在侵权风险（自由实施差距）和潜在许可或主张机会。
5. **组合合理化**：COCO生成分级建议：维护并积极起诉的资产、为防御价值而维护的资产、许可的资产和放弃以降低成本的资产。
6. **并购准备报告**：COCO生成知识产权尽职调查准备评估，识别买方尽职调查审查中会出现的差距，并建议在出售流程开始前的修复步骤。

使用该工作流的科技公司报告，通过系统性放弃非战略性资产，年度知识产权维护成本降低15%至25%。并购交易的知识产权尽职调查准备时间从6至8周缩短至2至3周。本会被收购方或诉讼发现的开源合规差距被主动识别和修复，保留了交易价值。

**受益群体：**
- **知识产权律师和专利律师**：管理大型专利组合，需要在有限预算内做出战略性起诉和维护决策。
- **首席法务官和总法律顾问**：必须向董事会提供知识产权组合健康状况和风险的清晰视图。
- **并购和企业发展团队**：需要在买卖双方交易中快速评估知识产权组合质量。
- **工程和产品负责人**：需要在发布前了解开源义务，并确保在竞争市场中的自由实施。

## 5个实用提示词

**提示词1 — 知识产权组合健康评估**
```
请分析我方知识产权组合并生成健康评估。组合数据如下。

对每个资产类别，识别：
1. 面临即时风险的资产（到期、失效、争议中）
2. 覆盖缺口（关键产品或市场缺乏知识产权保护）
3. 当前未记录的许可合规义务
4. 建议的即时行动，优先级：严重/高/中/低

专利组合：[列出状态、年费日期、受让人]
商标组合：[列出司法管辖区、续展日期]
关键软件组件：[列出开源许可证]
关键许可协议（流入/流出）：[描述]

我方关键产品/功能：[列出]
我方关键市场：[列出司法管辖区]
```

**提示词2 — 专利权利要求范围分析**
```
分析以下专利权利要求并评估：
1. 每项独立权利要求提供的保护范围（宽泛、中等、狭窄）
2. 我方产品[产品描述]是否似乎在每项权利要求的范围内
3. 任何竞争对手产品[竞争对手产品描述]是否似乎侵犯这些权利要求
4. 权利要求解释风险——可能被法院狭义解释的语言
5. 建议的起诉策略：按原样维护/提交延续申请/扩宽权利要求/放弃

专利号：[编号]
独立权利要求：[粘贴权利要求]
我方产品描述：[描述关键功能]
```

**提示词3 — 开源许可合规审计**
```
我方软件产品使用以下开源组件。对每个组件：
1. 识别许可证类型（MIT、Apache 2.0、GPL v2、GPL v3、LGPL、AGPL等）
2. 说明该许可证对我方施加的关键义务
3. 评估我方当前的分发方式是否触发copyleft义务
4. 识别我方正在一起使用的组件之间的任何许可证不兼容性
5. 标记我方可能不合规的组件并建议修复措施

开源组件：
[列出带有版本号和许可证标识符的组件]

我方分发方式：
[描述：如SaaS（不分发）、二进制分发、源代码分发]
```

**提示词4 — 自由实施风险评估**
```
我方正在计划推出以下新产品功能：[详细技术描述功能]。

请进行初步自由实施风险评估：
1. 识别我方功能采用的关键技术概念和方法
2. 列出与该技术最相关的专利分类代码（CPC/IPC）
3. 识别竞争对手[列出竞争对手]持有的任何可能相关的公开已知专利
4. 评估风险级别：高/中/低，附推理
5. 建议是否需要专利律师出具正式自由实施意见
6. 如风险为高，建议设计规避方案

功能描述：
[详细技术描述]
已知竞争对手专利（如有）：
[列出]
```

**提示词5 — 并购知识产权尽职调查准备**
```
我方正在为潜在的[出售/收购/投资]交易做准备。请进行知识产权尽职调查准备评估：

1. 权属链——识别关键知识产权资产的转让或职务发明文件中的任何缺口
2. 员工/承包商知识产权协议——标记任何可能在没有适当转让协议的情况下创造知识产权的员工或承包商
3. 第三方知识产权——识别嵌入我方产品中的任何第三方知识产权，可能需要我方尚未获得的许可
4. 开源风险——总结我方开源合规状况和任何已知差距
5. 负担——任何知识产权资产是否被质押为担保品、受留置权约束或受排他性许可覆盖？
6. 司法管辖区缺口——关键商标是否在我方有商业运营的所有司法管辖区注册？
7. 优先修复列表——5项如果不解决最可能降低交易价值或终止交易的问题

我方业务：[描述产品、市场、关键知识产权资产]
```
