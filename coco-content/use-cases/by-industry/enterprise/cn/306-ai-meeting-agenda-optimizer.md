# 使用案例 #306：AI 会议议程优化器

**角色**：运营经理 | **行业**：企业 | **任务**：日程安排

---

## 详细介绍

会议是现代企业运营中最不可或缺、同时也最容易造成浪费的活动之一。微软2023年的研究发现，员工平均将57%的工作时间花在会议和电子邮件上，但71%的高级管理人员认为大多数会议毫无成效。对于一家拥有500名员工的企业而言，结构不合理的会议每年可能造成超过200万美元的生产力损失。运营经理在这一问题上承受着不成比例的压力：他们需要跨部门协调，推动跨职能项目的一致性，却常常在一小时的同步会议结束后，没有明确的成果、负责人或截止日期。

问题的根源很少是努力不足——而是结构缺失。会议组织者往往在当天早上临时拼凑议程，没有考虑利益相关者的优先级、时间限制、所需决策的深度，以及参会者实际需要的准备材料。结果是：会议超时、重复讨论已定事项、未能暴露阻塞点、行动项目模糊不清。这种低效会产生连锁反应：用于澄清结果的跟进邮件、用于重新讨论未解决问题的额外会议，以及整个团队对会议文化的信任侵蚀。

COCO 通过充当智能议程设计伙伴，在每次会议开始前就完成结构化设计，从根本上改变这一局面。

**COCO 如何解决问题：**

1. **信息收集**：运营经理向 COCO 提供会议目的、参会者名单（含角色）、可用时间段以及相关背景信息——例如项目状态、待定决策或以往会议记录。

2. **议程架构**：COCO 分析输入信息，生成带有时间限制的结构化议程，明确标注每个议程项是决策点、信息分享还是工作讨论。当拟定范围超出可用时间时，COCO 会发出提示并建议推迟的内容。

3. **预读材料识别**：COCO 识别哪些参会者需要在会议前准备特定材料，并为每个角色起草简洁的预读摘要或问题提示，从而减少会议中花在背景介绍上的时间。

4. **主持人笔记**：针对每个议程项，COCO 生成主持要点、建议讨论问题和决策标准，为会议负责人提供清晰的脚本，在不压制真实对话的前提下保持讨论聚焦。

5. **行动项模板**：COCO 在议程中嵌入结构化的收尾环节，提示主持人为每项承诺行动记录负责人、截止日期和成功标准，防止出现"我们之后再跟进"的模糊状态。

6. **会后摘要草稿**：会议结束后，运营经理可以将原始笔记或会议记录粘贴给 COCO，COCO 将生成与原始议程结构对应的简洁摘要，几分钟内即可分发。

**可量化成果：**

实施结构化会议实践的组织报告，平均会议时长减少30-40%。使用AI辅助议程的团队，从决策到行动的周期时间缩短多达50%，因为行动项更加清晰且分配更明确。定期会议系列也在持续改进：COCO 可以参考以往会议摘要自动标记未解决的事项并延续跟进。运营经理报告每周节省3-5小时原本用于会议协调和跟进起草的时间。

**受益人群：**

- **运营经理**：减少会议协调的行政负担，在整个组织建立一致且高质量的会议文化。
- **项目负责人**：获得更清晰、以目标为导向的议程，帮助他们在不失控范围和时间的情况下主持跨职能评审。
- **高级管理人员**：减少在无定论会议中消耗的时间，将更多精力用于推动业务前进的决策。
- **所有与会者**：更充分地准备会议，更有意义地贡献，并在会议结束时对后续行动拥有更清晰的责任感。

## 实用提示词

**提示词 1 — 完整议程生成**
```
我需要主持一场关于[主题]的[60分钟/90分钟][每周/每月/临时]会议。参会者：[角色1]、[角色2]、[角色3]。必须达成的关键成果：[成果1]、[成果2]。背景信息：[简要背景或附上以往会议记录]。请生成带有时间限制的结构化议程，将每个议程项标注为"决策/讨论/信息"，并添加收尾行动项记录环节。
```

**提示词 2 — 预读材料包**
```
根据以下议程：[粘贴议程]，为每个参会角色起草预读信息。[角色1]应查阅或准备什么？[角色2]呢？每部分不超过150字，并以一个他们在会议中应准备好回答的具体问题作为结尾。
```

**提示词 3 — 议程审查与范围检查**
```
以下是一份[时长]分钟会议的议程草稿：[粘贴议程草稿]。请审查并告诉我：(1) 在可用时间内，范围是否现实？(2) 哪些议题可以异步处理？(3) 利益相关者可能提出但议程中缺失的内容是什么？如有需要，请提供修订版本。
```

**提示词 4 — 主持人脚本**
```
我将主持一场关于[主题]的会议，议程包含以下内容：[列出议程项]。针对每个议程项，请写出2-3条主持要点，以及一句当讨论偏题时可以使用的"停车场"提示语。参会者包括[角色/级别]。
```

**提示词 5 — 会后摘要**
```
以下是我今天[会议类型]会议的原始记录：[粘贴笔记或会议记录]。原始议程为：[粘贴议程]。请生成一份简洁的会议摘要，包含：(1) 已做出的关键决策，(2) 含负责人和截止日期的行动项，(3) 推迟至下次会议的议题，(4) 提出的风险或阻塞点。摘要不超过一页。
```
