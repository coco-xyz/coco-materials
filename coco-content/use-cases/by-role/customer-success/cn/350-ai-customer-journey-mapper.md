# #350 — AI 客户旅程映射助手

**角色**: 客户成功
**行业**: SaaS / 科技
**任务**: 分析
**标识符**: `ai-customer-journey-mapper`

---

## 简介

**痛点：客户流失事后可以预见，实时却无法察觉**

SaaS 公司的客户成功团队在一种使预防在结构上变得困难的信息架构下对抗客户流失。关于每个客户旅程的数据——完成的入职里程碑、采用的功能、打开和解决的支持工单、健康分数历史、季度业务回顾出席情况、关键联系人更换事件和续约历史——在组织的某处存在。但它以碎片形式存在：CSM 在 Salesforce 中的备注、项目管理工具中的入职任务、分析平台中的产品使用数据、工单系统中的支持历史以及 CS 平台中的健康分数。没有任何人拥有客户如何经历其与产品关系的单一连贯视图，而这种综合视图的缺失使得在干预还来得及之前识别预测流失的结构性模式几乎不可能。

旅程映射问题不仅仅是数据聚合问题——它是规模化的模式识别问题。管理 150 个客户的 CS 团队无法手动审查每个客户的完整旅程历史，以识别哪个旅程阶段与较高的流失风险相关、哪些入职里程碑未完成预测 6 个月后的低功能采用率，或者哪种支持工单量、参与模式和关键联系人稳定性组合创造了风险最高的客户档案。

客户旅程中的结构性缺口也不是均匀分布的。大多数 SaaS 产品有 2-3 个客户系统性地在其中挣扎的特定阶段——复杂的入职配置步骤、前 90 天的功能采用平台期，或者 6-9 个月后当初始热情消退而产品尚未深度嵌入工作流时的常见失败模式。这些结构性失败点可以通过模式分析来识别，但前提是有人在做分析。

留存客户与高流失客户旅程的比较是 CS 领导者可用的最强大诊断工具之一——也是最少被使用的工具之一。留存客户几乎无一例外地显示出与流失客户不同的旅程模式：前 60 天更高的里程碑完成率、更广泛的功能采用档案、更定期的高管参与、更快的首个用例实现价值时间。识别这些模式创造了一个预测模板：旅程偏离留存群体模式的客户是最紧迫需要主动干预的客户。

**COCO 如何解决这一问题**

COCO 帮助 CS 团队系统性地分析客户旅程数据——综合不同数据源、识别结构性缺口模式、比较高流失和留存客户路径，并生成有针对性的干预建议。

1. **旅程阶段定义和映射**：COCO 帮助 CS 团队定义完整客户旅程的各个阶段——从合同签署到入职、初始采用、扩展、续约和倡导——为每个阶段提供具体的里程碑定义、时间线预期和成功标准。
   - 设计带有里程碑完成基准的逐阶段旅程图
   - 识别客户最常停滞的阶段间交接点

2. **里程碑完成率分析**：COCO 分析客户完成哪些入职和采用里程碑、按什么顺序、以什么速度完成的历史数据——识别哪些里程碑未完成最强烈地预测下游流失。
   - 生成显示哪些里程碑被系统性跳过的里程碑完成热图
   - 识别区分高风险和低风险客户的里程碑完成模式

3. **流失与留存路径比较**：COCO 比较流失客户与留存客户的旅程模式——识别区分两个群体的具体旅程特征并生成预测风险档案。
   - 生成流失与留存群体的并排旅程比较报告
   - 识别最能预测最终流失的 3-5 个旅程变量

4. **流失阶段识别**：COCO 分析客户旅程中参与度系统性下降的地方——识别客户最可能脱离、停滞或开始离开旅程的具体阶段。
   - 用统计显著性指标按旅程阶段绘制流失风险图
   - 在数据支持的情况下将流失模式与特定产品或流程因素相连接

5. **客户级旅程健康评估**：COCO 综合可用数据生成个别客户旅程健康档案——将每个客户的当前旅程位置和完成率与其群体的基准进行比较。
   - 生成带有偏差标记的客户级旅程评分卡
   - 根据旅程健康严重性生成优先干预列表

6. **按旅程阶段设计干预措施**：COCO 根据模式分析为每个旅程阶段推荐具体干预措施——识别哪些 CS 行动、产品变更或流程改进将最有效地解决已识别的结构性缺口。
   - 基于流失分析设计阶段特定的剧本补充
   - 推荐风险阶段主动外联的时间和内容

**可量化的成果**

- **流失预测准确性**：系统性旅程分析将 CS 团队识别风险客户的时间比基于续约通知的被动方法提前 60-90 天
- **入职完成率**：在已识别流失阶段设计有针对性干预的团队将里程碑完成率提高 30-40%
- **实现价值时间改善**：识别并消除旅程结构性瓶颈将平均客户首次实现价值时间减少 25-35 天
- **干预针对性效率**：使用旅程模式数据优先主动外联的 CS 团队，与基于直觉的优先排序相比，报告干预成功率提高 2 倍
- **毛收入留存率**：实施系统性旅程监控的组织在 12-18 个月内毛收入留存率平均提升 6-8 个百分点

**受益角色**

- **客户成功经理**：管理 20-50 个客户，需要一种系统化方法来了解每个客户的旅程健康状况，而无需在多个系统中手动编制全貌
- **客户成功副总裁和总监**：设计 CS 团队战略，需要关于客户旅程中哪些地方团队工作产生最大杠杆效应的模式级洞察
- **产品团队**：需要了解哪些产品旅程摩擦点导致采用缺口和流失，以便优先考虑具有最高留存影响的改进
- **营收运营团队**：负责预测和毛收入留存报告，需要关于客户组合中流失风险趋势的早期、数据驱动信号

## 实用提示词

**提示词 1 — 客户旅程阶段定义**
```
我需要为我们的 [B2B SaaS 产品——描述：产品类别、典型客户规模、典型用例] 建立一个正式的客户旅程图，供我们的 CS 团队用于跟踪客户健康状况和指导干预时机。

我们目前对客户生命周期的理解：
- 合同签署 → 入职 → [按您的理解添加阶段]
- 平均合同价值：[$X]
- 平均合同期限：[12/24/36 个月]
- 从合同到上线的典型时间：[X 周]
- 我们团队当前最大的关注点：[您最常见到的失败模式是什么？]

请帮我定义：
1. 从合同签署到续约/流失决策的完整旅程阶段综合列表——每个阶段的清晰定义
2. 每个阶段的：进入标准、成功标准、预期时间线和退出标准（什么将客户推进到下一阶段）
3. 每个阶段内表明健康进展的 3-5 个最关键里程碑
4. 每个阶段表明客户面临停滞或流失风险的预警信号
5. 每个预警信号触发的建议 CS 团队行动
```

**提示词 2 — 流失与留存旅程模式分析**
```
我想了解哪些旅程特征区分了我们的流失客户和留存客户。我有过去 [18 个月] 中 [X] 个流失客户和 [Y] 个留存客户的数据。

每个客户可用的数据：
- 入职里程碑完成情况：[列出里程碑以及每个里程碑是否完成，以及何时完成]
- 功能采用：[使用了哪些功能、何时首次使用、使用频率]
- 支持工单：[数量、严重性和解决时间]
- 季度业务回顾出席情况：[参加/跳过——由谁]
- 关键联系人更换：[是/否、时机]
- 健康分数历史：[月度或季度分数]
- 合同价值和扩展历史

请：
1. 分析流失与留存群体，识别与流失最强相关的旅程特征
2. 计算每个特征的预测价值——如果 X 为真，客户流失的可能性增加多少？
3. 构建复合流失风险档案：哪种特征组合描述了风险最高的客户档案？
4. 识别旅程中可以有把握预测未来流失的最早节点
5. 建议干预时间线：在什么旅程阶段以及基于什么信号，CSM 应该触发主动留存行动？
```

**提示词 3 — 入职流失分析**
```
我们的入职完成数据显示，客户未能完成几个关键里程碑，但我不知道哪些未完成对下游留存影响最大。以下是我们的入职里程碑完成数据：

里程碑 1：[名称]——[X%] 完成率——平均完成时间：[X 天]
里程碑 2：[名称]——[X%] 完成率——平均完成时间：[X 天]
里程碑 3：[名称]——[X%] 完成率——平均完成时间：[X 天]
[继续所有里程碑]

完成所有里程碑的客户：[X%] 在首次续约时续签
完成 <50% 里程碑的客户：[X%] 在首次续约时续签

请：
1. 识别哪些里程碑未完成与差的续约率最强相关
2. 对于影响最大的里程碑，假设是什么阻止了完成——产品复杂性、流程摩擦、客户资源限制、不清晰的价值，还是 CSM 流程缺口
3. 推荐对于接近每个高风险里程碑的客户具体应该有什么不同——在 CSM 主动行动、产品指导或流程变更方面
4. 设计里程碑救援剧本：当某个客户错过关键里程碑超过预期完成日期 [X 天] 时，CSM 应该做什么？
5. 估算如果前 3 个里程碑的完成率提高 20 个百分点，潜在的续约率改善
```

**提示词 4 — 个别客户旅程健康评估**
```
我需要为我担心的一个客户建立旅程健康评估。请帮我分析他们的旅程并识别干预优先事项。

客户：[公司名称、行业、规模]
合同详情：[价值、开始日期、续约日期、合同期限]
用例：[描述他们购买的内容和原因]

旅程数据：
- 入职：[哪些里程碑已完成，哪些未完成，时间]
- 功能采用：[他们正在使用什么，没有使用什么，趋势]
- 支持历史：[工单量、反复出现的主题、任何重大事件]
- 参与度：[季度业务回顾出席情况、CSM 会议频率、高管参与程度]
- 关键联系人状态：[他们的关键联系人还在职吗？最近有任何变化吗？]
- 健康分数历史：[最近趋势]
- 上次有意义的积极互动：[何时，什么]
- 我注意到的红旗：[描述]

请：
1. 评估此客户的旅程健康状况，相对于该任期和阶段的健康客户应该是什么样子
2. 识别其旅程档案中 2-3 个最重要的风险因素
3. 生成关于为何表现不佳的假设——什么最可能驱动风险？
4. 推荐具体干预计划，包括：本周的立即行动、30 天计划以及我需要回答以确定续约可能性的关键问题
5. 起草我可以用来开启恢复对话的再参与邮件开场或通话议程
```

**提示词 5 — 客户组合级旅程健康报告**
```
我需要为我的 CS 总监准备一份客户组合健康报告，显示我 [X] 个客户中旅程健康的分布情况。请帮我构建此报告。

客户组合摘要：
- 总客户数：[X]
- 管理的总年经常性收入：[$X]
- 按续约季度划分的客户数：[Q1: X, Q2: X, Q3: X, Q4: X]
- 客户层级分布：[企业: X, 中市场: X, SMB: X]

按客户划分的旅程健康评估（我会提供此数据）：
[对于每个客户，或按层级分组：里程碑完成百分比、功能采用广度、参与频率、健康分数、距下次续约的时间]

请构建包含以下内容的报告：
1. 客户组合健康摘要：我的客户组合中有多少百分比处于绿色/黄色/红色健康状态，按类别风险的年经常性收入是多少？
2. 结构性模式：是否有多个客户同时在特定旅程阶段挣扎？
3. 优先干预列表：哪些客户需要立即关注，哪些需要在未来 30 天内主动外联，哪些进展良好？
4. 风险加权续约预测：基于旅程健康模式，每个季度群体的现实续约预测是什么？
5. 给我的 CS 总监的建议：为有效管理最高风险客户，我需要哪些资源或支持？
```
