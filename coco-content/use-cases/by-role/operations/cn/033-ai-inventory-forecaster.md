# Use Case #033: AI Inventory Forecaster

**Role**: Supply Chain Manager / Demand Planner / Operations Director | **Industry**: E-commerce, Retail, Manufacturing, Distribution, CPG | **Task**: Demand Forecasting, Inventory Optimization, Stockout Prevention, Overstock Reduction

---

## 社媒文案

去年黑色星期五，我们囤了40000件畅销产品。周六下午2点就卖完了。整个剩余周末估计损失了28万美元的销售。与此同时，我们积压了45万美元的慢动销库存——来自需求规划师高估了60%的产品线。

两个错误，同一个根因：我们的预测基于去年数据+增长系数+直觉。没有考虑到缺货的竞品（把需求推向了我们）、为某个SKU走红的TikTok视频、或者消费者偏好从我们积压品类转移的宏观变化。

用电子表格做库存规划就像带着刀去打一场数据战。

COCO的AI Inventory Forecaster用信号检测取代了直觉：
- 分析每个SKU的36个月销售数据、季节性模式和趋势曲线
- 整合外部信号：竞品库存状态、社交媒体提及、天气预报、经济指标
- 在SKU级别生成带置信区间的需求预测（不只是一个单一数字）
- 推荐平衡缺货风险和库存持有成本的最优补货点和数量
- 标记异常："SKU #4721趋势比预测高340%——调查1月12日的TikTok提及"

缺货事件：下降67%。积压减值：下降43%。库存周转率：从4.2倍提升到6.1倍。释放的营运资金：120万美元。我们不再靠猜了。

## 详细介绍

**痛点：用电子表格做库存预测在缺货和积压上损失数百万**

库存管理是一个两面失败都很昂贵的平衡之术。缺货意味着收入损失、客户失望、市场份额送给竞品。积压意味着营运资金被占用、仓储成本、打折和减值。两者之间的最优点需要准确的需求预测——而这正是大多数公司失败的地方。

传统预测依赖于用增长系数和规划师直觉调整的历史销售数据。这种方法错过了不出现在历史数据中的需求信号：病毒式社交媒体时刻、竞品缺货、天气驱动的需求变化、宏观经济变化和促销日历效应。

犯错的代价是惊人的。IHL Group估计，全球零售商每年因积压和缺货的综合情况损失1.75万亿美元。对于一家收入5000万美元的中型电商公司，预测误差通常代表200-500万美元的销售损失和减值。用Excel的需求规划师正在用不充分的工具尽力应对一个越来越不可预测的市场。

**COCO如何解决**

COCO的AI Inventory Forecaster将历史分析与实时信号检测相结合，进行SKU级别的需求预测。

1. **多变量需求建模**：远超"去年+增长系数"：
   - 分析每个SKU的24-36个月销售历史
   - 将时间序列分解为趋势、季节性和周期性组件
   - 考虑促销、定价变更和产品生命周期阶段
   - 模拟蚕食效应（新产品发布从现有SKU抢走销量）
   - 使用类似产品和市场数据处理新产品预测

2. **外部信号集成**：整合电子表格无法处理的数据：
   - 竞争情报：竞品价格变化、库存可用性、促销活动
   - 社交媒体趋势：病毒式提及、网红帖子、话题标签速度
   - 天气数据：影响季节性产品的温度和降水预报
   - 经济指标：消费者信心、就业数据、通胀趋势
   - 行业报告：品类增长数据、市场份额变化
   - 日历效应：节假日、活动、学校时间表、文化纪念日

3. **概率预测**：用风险感知范围替代单一数字预测：
   - 提供带80%置信区间的需求预测（低/预期/高）
   - 实现风险加权的库存决策（关键SKU备货到第80百分位，低利润率的备到第50）
   - 量化每个SKU的预测不确定性（某些产品本质上更可预测）
   - 用蒙特卡洛模拟进行高峰期规划（黑色星期五、节假日）

4. **补货优化**：计算最优库存参数：
   - **补货点**：何时下新订单（考虑提前期和需求变异性）
   - **补货数量**：订多少（平衡订购成本vs持有成本）
   - **安全库存**：达到目标服务水平所需的缓冲库存
   - **动态调整**：参数随需求模式变化自动更新
   - **供应商提前期建模**：考虑供应商交付时间的变异性

5. **异常检测和早期预警**：在需求变化成为问题之前捕获：
   - 对照预测实时监控销售速度
   - 当实际值显著偏离预测时自动警报
   - 根因假设："SKU #4721趋势比预测高340%——可能原因：1月12日TikTok提及（14.5万次观看），1月10日检测到竞品缺货"
   - 实现快速响应：紧急补货、替代品规划、需求引导

6. **假设分析**：模拟商业决策对库存需求的影响：
   - "如果我们对这个品类做八折促销会怎样？"
   - "如果供应商A的提前期从4周增加到8周会怎样？"
   - "如果我们推出产品B——它如何蚕食产品A？"
   - "如果我们扩展到3个新的地理市场会怎样？"
   - 帮助领导层做出考虑库存的商业决策

**可量化的结果**

- **缺货减少**：缺货事件减少67%
- **积压减少**：减值降低43%
- **库存周转率**：从4.2倍升至6.1倍
- **预测准确度（MAPE）**：从32%改善至14%
- **释放的营运资金**：通过优化库存水平释放120万美元
- **保护的收入**：每个旺季28万美元以上的预防销售损失
- **规划师生产力**：手动预测时间减少60%，更多时间用于战略规划
- **供应链响应能力**：需求变化提前2-3周检测到

**受益角色**

- **需求规划师**：更好的工具替代直觉；重心从表格维护转向战略分析
- **供应链经理**：更少的缺货和积压；更平稳的运营；更好的供应商关系
- **CFO/财务**：释放的营运资金；更低的库存减值；更好的现金流可预测性
- **销售团队**：客户想要时产品有库存；更少的"抱歉，缺货"时刻
- **仓库/物流**：更可预测的入库量；更好的空间和劳动力规划
- **客户**：更好的产品可用性；更少的延期交货和取消

## 实用提示词

**提示词 1: 生成需求预测**
```
为未来[3/6/12个月]在SKU级别生成需求预测。

历史销售数据（过去24-36个月）：
[粘贴按SKU的月度销售或描述数据可用性]

额外背景：
- 即将到来的促销：[列出计划的促销及日期]
- 价格变更：[任何计划的价格调整]
- 新品发布：[可能蚕食或互补的产品]
- 已知供应约束：[任何供应链问题]
- 季节性事件：[双十一、年货节、暑假等]

对每个SKU提供：
1. 月度需求预测含低/中/高场景
2. 置信区间（80%）
3. 关键假设
4. 建议的安全库存水平
5. 补货点和数量
6. 高预测不确定性SKU的标记
```

**提示词 2: 库存健康审计**
```
审计我们当前库存以寻找优化机会。

当前库存：
[粘贴库存数据：SKU、在手数量、单位成本、月均销量、供应天数]

分析并识别：
1. **积压**（>90天供应）：哪些SKU有超额？预估持有成本？
2. **缺货风险**（高周转项目<14天供应）：哪些SKU需要紧急补货？
3. **死库存**（90天内<1件售出）：绑定在不动销库存中的价值？
4. **ABC分类**：按收入贡献分类SKU（A=前80%、B=接下来15%、C=最后5%）
5. **补货优先级**：本周需要补货的SKU排名列表
6. **清仓候选**：考虑打折或减值的SKU
7. **营运资金机会**：通过优化可以释放多少资金？
```

**提示词 3: 供应链中断场景规划**
```
帮我规划潜在的供应链中断及其库存影响。

当前供应链：
- 关键供应商：[列出供应商、产品、提前期、地理位置]
- 当前库存水平：[按产品类别或关键SKU]
- 月需求：[按类别的月均销量]
- 替代供应商：[列出备选供应商及其能力]

模拟这些场景：
1. **供应商延迟**：主要供应商提前期从[X]增加到[Y]周。对缺货的影响？建议的安全库存调整？
2. **需求激增**：[类别/SKU]需求因[原因]增加[X]%。当前库存和供应管线能否应对？
3. **物流中断**：从[地区]的发货延迟[X]周。哪些SKU风险最大？替代采购选项？
4. **原材料短缺**：关键零部件稀缺，供应商产能降低[X]%。分配策略？

对每个场景：
- 财务影响（销售损失、加急成本、持有成本）
- 现在建议的预防措施
- 执行应急计划的触发点
- 给销售/市场团队的沟通计划
```
