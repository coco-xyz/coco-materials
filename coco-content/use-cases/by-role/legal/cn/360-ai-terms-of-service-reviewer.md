# #360 — AI服务条款审查器

**角色**: 法务
**行业**: SaaS/科技
**任务**: 合规
**Slug**: ai-terms-of-service-reviewer

---

## 简介

SaaS公司的生死与服务条款（ToS）息息相关。对于供应商而言，ToS是界定可接受使用范围、限制责任、保护知识产权、确立付款条款和规范争议解决的核心法律文件。对于买方和企业客户而言，在采购审批前审查供应商ToS是供应商风险管理的必要步骤——然而大多数法务团队的审查缺乏充分的结构性和一致性。

问题的规模不容小觑。一家审查企业软件采购的典型SaaS公司每年可能需要评估30至60份供应商ToS文件，每份文件包含2000至20000字、15至40个章节。技术交易律师进行彻底审查需要每份文件2至5小时，按每小时3000至5000元计算，仅ToS审查每年的法律费用就达6万至25万元——谈判尚未开始。

ToS中的关键风险领域已广为人知，但在截止日期压力下经常被遗漏：允许供应商无需通知即单方面修改条款的条款；不包含间接损失赔偿但强制客户承担无限数据损失责任的责任上限；赋予供应商对客户数据或客户生成内容广泛权利的知识产权转让条款；强制仲裁和集体诉讼豁免条款；通知期不足的自动续约条款；以及措辞宽泛到足以为常规情况下的不履行提供免责保护的不可抗力条款。

COCO通过结构化的审查工作流解决这一问题：

1. **ToS摄入**：律师上传ToS文件并说明审查角度（买方审查供应商ToS，或供应商审查自身ToS的市场适应性）。
2. **风险分类**：COCO针对已知风险模式库分析每个条款——单方修改、无限责任、知识产权转让、仲裁、自动续约陷阱、数据权利——并将每种风险定性为严重、高、中或低级别。
3. **市场基准比较**：COCO将关键条款与相关SaaS细分市场的市场标准进行比较，标记异常条款并解释市场标准条款的样貌。
4. **谈判手册**：对于买方审查，COCO起草谈判备忘录，识别需要反驳的条款、请求什么替代语言以及最低可接受立场。
5. **供应商ToS健康检查**：对于供应商自审，COCO评估ToS是否能通过企业采购审查，标记企业法务团队经常拒绝的条款，并建议更符合市场标准的替代方案。
6. **摘要报告**：COCO生成一页纸的决策备忘录：按原样推进、协商修改后推进，或上报/拒绝。

使用该工作流的团队将ToS审查时间从每份文件3小时缩短至35分钟，使法务团队以相同资源处理5倍以上的审查量。当法律审查加速后，软件采购的采购周期缩短30%至40%。使用COCO市场基准更新ToS的企业供应商报告，在销售周期中来自潜在客户的法律反驳减少25%。

**受益群体：**
- **技术交易律师**：代表企业买方审查SaaS、云和软件供应商协议。
- **采购和供应商管理团队**：需要比律师排队时间更快获得法律风险评估。
- **SaaS创始人和产品负责人**：需要其ToS通过企业采购审查，同时不吓跑客户。
- **合规和供应商风险管理人员**：跟踪活跃供应商关系组合中的ToS合规状态。

## 5个实用提示词

**提示词1 — 完整ToS风险审查（买方视角）**
```
请从[公司名称]作为客户/买方的角度审查以下供应商服务条款。我方是[行业]领域的[公司类型]。

对每个章节，请说明：
- 风险级别：严重/高/中/低/可接受
- 产生风险的具体语言（引用原文）
- 风险及其商业影响的通俗说明
- 这是市场标准条款还是异常条款
- 建议的谈判立场或退让语言

特别关注：
- 单方修改权
- 责任上限和排除（尤其是数据损失）
- 知识产权和数据权利
- 自动续约和终止条款
- 争议解决和仲裁条款
- 不可抗力条款

ToS文本：
[在此粘贴ToS]
```

**提示词2 — 供应商ToS自评**
```
我正从企业客户法务团队的角度审查我方公司的服务条款。请识别：
1. 企业法务团队通常拒绝或要求修改的条款
2. 比[SaaS类别]供应商市场标准更为严格的条款
3. 企业买方无法接受的责任条款
4. 可能阻碍企业采购的知识产权和数据权利条款
5. 需要更新以提升企业销售转化率的具体章节

对每个问题，建议市场标准的替代语言。

我方ToS：
[在此粘贴ToS]
```

**提示词3 — 特定条款谈判准备**
```
我需要就供应商ToS中的以下条款进行谈判。供应商是向我方公司提供[服务]的[供应商类型]。我方是[买方/客户]。

请准备谈判备忘录：
1. 分析现有条款并说明其对我方造成的风险
2. 说明我方首选替代语言及其更为平衡的原因
3. 提供"底线"最低可接受立场
4. 预测供应商可能的反对意见并起草反驳论点
5. 建议这是否为交易障碍或可谈判的让步

条款：
[在此粘贴条款]
```

**提示词4 — ToS比较：供应商与市场标准**
```
将以下供应商ToS条款与[SaaS类别]协议的市场标准条款进行比较。对每项条款，说明：
1. 供应商当前语言（摘要）
2. 该条款的市场标准
3. 供应商偏离市场标准的程度（轻微/中等/显著）
4. 偏差是否有利于供应商或客户
5. 建议的谈判方法

待比较的ToS条款：
[列出关键条款或粘贴完整ToS]
```

**提示词5 — 新产品功能ToS更新**
```
我方SaaS平台推出了以下新功能：[描述功能]。

请审查我方现有服务条款并：
1. 识别与新功能相冲突或未能涵盖新功能的条款
2. 起草新的或更新的ToS语言以解决该功能问题（可接受使用、知识产权、数据处理、责任）
3. 标记新功能在现有条款下改变我方法律风险敞口的条款
4. 识别在现有用户使用该功能前可能需要从其获得的新同意或披露

现有ToS：
[粘贴ToS]

新功能描述：
[详细描述]
```
