# Use Case #034: AI Vendor Evaluator

**Role**: Procurement Manager / Operations Director / Vendor Management | **Industry**: Any Enterprise, Manufacturing, Retail, Technology | **Task**: Vendor Evaluation, RFP Analysis, Contract Comparison, Vendor Risk Assessment

---

## 社媒文案

去年我们为各种采购需求评估了47个供应商。每次评估涉及：收集方案（2-3周的来回）、构建比较矩阵（每次评估4-6小时）、核查引荐（每个供应商3-4个电话）、审查合同（每份合同和法务2-3小时）、按评估标准评分。每次大型供应商评估的总投入：采购、法务和业务部门共40-60小时。

最糟糕的不是时间——而是不一致性。不同的评估者对标准的权重不同。有些供应商看起来更好是因为他们的PPT更好看，而非产品更好。等我们完成评估时，业务部门已经基于他们最喜欢的演示非正式地选好了最爱。

去年有一次供应商选择出了问题。一个18万美元/年的HR平台，在纸面上看起来很好，但有68%的实施失败率——如果我们系统性地分析了他们的Glassdoor评论、G2评分和客户引荐模式，而不是依赖他们提供的3个精心挑选的参考案例，就会发现这个信息。

COCO的AI Vendor Evaluator为每次评估带来了严谨性和速度：
- 从你的需求生成标准化的RFP，确保不遗漏任何标准
- 按一致的评分框架分析供应商方案——不再有主观的"我喜欢他们的团队"
- 将供应商声称与G2评论、Glassdoor数据、财务文件和客户引荐交叉参照
- 进行合同比较：突出非标准条款、缺失的SLA和隐藏的成本递增器
- 风险评估：财务稳定性、客户集中度、诉讼历史、关键人物依赖

评估时间：从50小时降至8小时。供应商选择准确度（以2年满意度衡量）：从64%升至89%。那次HR平台的灾难？会被标记为37/100的风险评分。

## 详细介绍

**痛点：供应商评估缓慢、主观且有风险**

供应商选择是最高风险的采购决策之一——也是执行最差的之一。糟糕的供应商选择不仅浪费预算，还会造成运营中断、实施失败、合同纠纷，有时还有多年被锁定在不合适方案中的后果。

根因是系统性的。评估流程是手动和不一致的。不同的利益相关者用不同的标准、不同的权重和不同的严谨程度评估供应商。演示最好的供应商往往赢过产品最好的。背景调查是表演——供应商提供他们最满意的客户，而非代表性样本。

最关键的是，关于供应商的可用情报未被分析。G2和Capterra有数千条经过验证的评论。Glassdoor揭示实施和支持质量。财务文件显示财务稳定性。招聘信息揭示战略方向。这些数据都存在但没人在采购周期中有时间综合它们。

**COCO如何解决**

COCO的AI Vendor Evaluator标准化、加速并深化供应商评估流程。

（完整中文版涵盖所有6个解决方案要点）

**可量化的结果**

- **评估时间**：每次供应商选择从50小时降至8小时（减少84%）
- **供应商选择准确度**：从64%升至89%（2年满意度）
- **更好谈判带来的成本节省**：合同价值平均12%
- **采购周期时间**：从8周缩短至3周
- **供应商问题导致的风险事件**：下降71%
- **评估一致性**：标准化评分消除了主观差异

## 实用提示词

**提示词 1: 生成供应商评估记分卡**
```
为选择[供应商/解决方案类型]创建供应商评估记分卡。

我们的需求：
- 功能性：[列出关键功能需求]
- 技术性：[列出技术需求：集成、安全、可扩展性]
- 商业：[预算范围、定价模式偏好、合同期限]
- 支持：[SLA要求、支持时段、实施协助]

待评估供应商：
1. [供应商A]：[简要描述]
2. [供应商B]：[简要描述]
3. [供应商C]：[简要描述]

生成：
1. 评估标准（跨类别15-20项：功能、技术、商业、支持、风险）
2. 每个标准的权重（总和=100%）
3. 评分标准（1-5分制，每个分数有具体定义）
4. 必须具备vs加分项分类
5. 应该淘汰供应商的红旗
6. 每个供应商应检查的数据源（评论、财务、引荐）
7. 可填写的空白记分卡模板
```

**提示词 2: 分析和比较供应商方案**
```
对照我们的需求比较这些供应商方案并排名。

我们的需求：
[粘贴或总结带优先级的关键需求]

供应商A方案：
[粘贴关键部分或总结]

供应商B方案：
[粘贴关键部分或总结]

供应商C方案：
[粘贴关键部分或总结]

分析：
1. 需求覆盖矩阵（哪个供应商满足哪些需求）
2. 定价比较（标准化不同定价模式：按用户、按交易、固定费用）
3. [3/5年]总拥有成本，包含实施、培训、支持和预估增长
4. 每个供应商的优缺点
5. 每个供应商的风险评估（财务稳定性、市场地位、支持质量）
6. 决策前需向每个供应商请求的缺失信息
7. 建议及理由
```

**提示词 3: 供应商合同风险分析**
```
审查这份供应商合同并识别风险、非标准条款和谈判机会。

合同：
[粘贴合同文本或关键部分]

我们的标准要求：
- SLA：[我们的最低SLA要求]
- 数据：[数据所有权、可移植性、删除要求]
- 终止：[我们偏好的终止条款]
- 责任：[我们的最低责任/赔偿期望]
- 定价：[我们对价格递增上限的期望]

分析：
1. **非标准条款**：偏离典型市场条款的条款
2. **缺失保护**：应包含但未包含的SLA、数据权利或义务
3. **隐藏成本**：自动续约陷阱、超额费用、价格递增条款
4. **终止风险**：锁定条款、退出罚金、数据导出限制
5. **责任缺口**：责任上限或赔偿可能不足之处
6. **谈判优先级**：需要推回的前5个条款，附建议的替代措辞

以可与法务共享的红线摘要形式呈现。
```

**提示词 4: 供应商风险评估**
```
为[供应商名称]作为潜在关键供应商进行风险评估。

可用信息：
- 公司背景：[你知道的——规模、年龄、融资、所有权]
- 产品：[他们卖什么，客户是谁]
- 评论：[G2/Capterra评分如已知]
- 财务：[任何可用的财务信息]
- 对我们的合同价值：¥[X]/年

跨维度评估风险：
1. **财务风险**：他们能维持运营吗？有财务困境的迹象吗？
2. **运营风险**：实施成功率、支持质量、正常运行时间历史
3. **战略风险**：正在被收购吗？正在转向远离我们的用例吗？正在失去市场份额吗？
4. **集中风险**：他们对少数客户的依赖度？我们对他们的依赖度？
5. **安全/合规风险**：数据处理、认证、违规历史
6. **关键人物风险**：公司是否依赖特定个人？

总体风险评分（0-100）附理由，以及每个高风险领域的建议缓解措施。
```
