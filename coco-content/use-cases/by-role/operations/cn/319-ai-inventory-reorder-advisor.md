# 用例 #319：AI 库存补货顾问

**角色**：运营经理 | **行业**：企业、制造、物流、零售 | **任务**：分析、库存管理、规划

---

## 详细介绍

**痛点：库存补货决策被动、凭感觉，双向代价高昂**

库存管理处于两种灾难性失败模式的交汇点：导致生产中断或客户失望的缺货，以及占用资金、消耗仓储空间并面临报废风险的积压库存。大多数运营团队同时承受着这两种失败——某些SKU断货的同时，另一些却积压过多——因为驱动库存水平的补货决策是被动且不一致地做出的。一位普通运营经理负责管理数百甚至数千个SKU，每个SKU都有截然不同的需求模式、供应商交货期和重要性级别，却使用基于静态公式构建的电子表格来做出补货决策，这些公式根本无法适应实际消耗的动态现实。

根本问题在于，良好的库存补货决策需要同时综合多个变量——而且是持续进行，而非仅在月度计划会议上。任何给定SKU的需求都会季节性波动，对促销活动和市场事件做出响应，并以简单移动平均线无法捕捉的方式受到下游生产计划的影响。供应商交货期不是常数，它们是受供应商产能、货运中断和全年订单量变化驱动的分布值。将交货期视为固定数值的安全库存计算，在高波动期系统性低估了库存风险——而这恰恰是缺货最具破坏性的时候。大多数组织确实拥有进行这种分析所需的数据——存在于ERP系统、供应商门户和需求规划工具中——但将这些数据综合为可执行的补货决策，需要运营团队根本没有精力为每个SKU单独完成的分析工作。

供应商可靠性增加了大多数组织完全依靠非正式知识处理的维度。有经验的采购员知道哪些供应商总是提前交货、哪些在交货期上有水分、哪些存在影响可靠性的季节性产能限制——但这些知识存在于人们的脑海中，没有被系统性地记录，而且当采购员换岗时就会流失。一个拥有200家活跃供应商的组织，可能有40家之间存在有意义的可靠性差异，这些差异应该直接影响安全库存的计算，却没有结构化的方式将这些信息纳入补货决策。结果是安全库存水平依据经验法则而非供应商特定风险评估来设定。

需求季节性和促销提升给静态模型带来了额外的复杂性，而静态模型对此处理得很差。进行促销活动的零售商可能会看到特色商品3至5倍于正常需求速度的峰值，需要在供应商交货期之前数周就开始提前建立库存。向汽车OEM供货的制造商可能在夏季车型切换期间看到需求下降40%，如果不调整补货节奏就会面临库存积压风险。这些都是可预测的周期性规律，应该系统性地驱动补货调整——但将它们纳入基于电子表格的规划需要每个周期手动干预，而手动流程在大型SKU组合中的执行并不一致。

**COCO 如何解决这一问题**

COCO 通过帮助运营团队分析消耗模式、模拟需求场景、评估供应商表现、计算情境适当的补货参数并生成考虑真实库存环境全部复杂性的可执行建议，来支持库存补货决策制定。

1. **消耗模式分析**：COCO 处理历史使用数据，识别需求模式、趋势线和应影响补货计算的异常情况。
   - 按SKU计算平均需求速度、变异系数和季节性指数
   - 标记需求高波动性需要更高安全库存缓冲的品项

2. **交货期和供应商可靠性评估**：COCO 分析供应商交货期数据以评估实际与承诺的交付表现。
   - 按供应商计算平均交货期、交货期标准差和准时交付率
   - 生成供应商风险分级，为安全库存和订单时机建议提供依据

3. **补货点和补货量计算**：COCO 生成基于统计的补货点和经济订货量计算，考虑需求变动性和供应商可靠性。
   - 根据服务水平目标和交货期变动性应用适当的安全库存公式
   - 生成包含可见假设条件的补货建议供采购员审核

4. **需求场景模拟**：COCO 帮助模拟即将到来的事件——促销活动、季节性峰值、生产计划变更——应如何影响库存建立计划。
   - 估算预期需求提升所需的库存要求，附时间和数量指导
   - 标记需要在活动前采取库存行动的品项及建议订货日期

5. **积压和报废风险识别**：COCO 识别当前或计划库存水平超过预计消耗的SKU，标记积压库存风险。
   - 计算滞销品项的供应周数并估算持有成本
   - 针对积压库存情况推荐减少库存的策略

6. **补货行动摘要生成**：COCO 将分析综合为按优先级排序、可执行的补货建议，供采购员审核和执行。
   - 按紧急程度和风险级别排名生成每日或每周补货行动清单
   - 生成为审计和审核目的支撑补货决策的文档

**可量化的成果**

- **缺货频率**：当补货点纳入需求变动性和供应商交货期变化时，缺货事件减少30-50%
- **积压库存水平**：应用数据驱动补货模型的运营团队通常在两个库存规划周期内减少积压库存持有成本15-25%
- **补货分析时间**：每位采购员每个规划周期手动分析3-5小时 → 审核和例外处理缩短至30-45分钟
- **供应商交货期风险**：按供应商了解交货期变化有助于对可靠供应商减少安全库存过度配置20-35%
- **库存周转率**：改善的补货时机和数量纪律通常推动库存周转率提升10-20%

**受益角色**

- **运营经理**：无需手动审查每个SKU即可获得组合层面的库存风险和行动优先级可见性，从而专注于例外情况管理
- **库存规划员/采购员**：收到有数据支撑的补货建议和分析，而非依赖静态电子表格触发器，从而更快、更自信地做出决策
- **供应链经理**：系统性地了解供应商可靠性规律，从而更好地确定供应商发展优先级和合同谈判策略
- **财务与会计**：得益于优化后的订货模式，库存水平更加可预测，持有成本降低，现金流预测更加准确

## 实用提示词

**提示词 1：分析SKU需求模式并计算补货参数**
```
请分析以下SKU的需求历史记录，计算适当的补货参数，包括补货点、安全库存和经济订货量。

SKU数据：
[粘贴SKU列表及月度历史需求——格式：SKU | 月份 | 消耗数量]

每个SKU的供应商数据：
- 供应商名称：[名称]
- 承诺交货期：[天数]
- 过去12个月实际交货期：[实际交货期列表]
- 单位成本：[成本]
- 最低订货量：[MOQ]
- 订单处理成本（内部）：[成本]
- 年度持有成本率：[百分比]

服务水平目标：[百分比，如95%或98%]

对每个SKU计算并输出：
1. 月均需求及需求标准差
2. 需求变异系数（CV）及解读（低/中/高变动性）
3. 供应商交货期：均值、标准差、准时交付率
4. 安全库存计算，附公式和假设条件说明
5. 补货点（以数量和供应天数表示）
6. 经济订货量
7. 建议补货频率
8. 风险分级：高/中/低缺货风险
```

**提示词 2：生成每周补货行动清单**
```
请根据当前库存情况和计算出的补货参数，生成按优先级排序的补货行动清单。

当前库存快照（截至[日期]）：
[粘贴库存数据——格式：SKU | 描述 | 当前库存（数量）| 补货点 | 日均需求 | 交货期（天）| 在途采购订单（数量，预计到货日期）]

紧急程度分级：
- 关键：当前库存低于安全库存水平
- 紧急：当前库存低于补货点且无在途订单
- 需要行动：当前库存将在[X]天内达到补货点
- 监控：库存高于补货点但持续下降

输出内容：
1. 需要立即下采购订单的关键品项（列表附数量和推荐供应商）
2. 本周需要下订单的紧急品项（列表附建议数量和订货日期）
3. 本周需要规划的待行动品项，附预计补货触发日期
4. 接近补货点的监控品项——标记以备下次审查
5. 积压库存品项（超过[X]周供应量）——建议暂停订货或减少订量
6. 摘要指标：按紧急程度分类的SKU总数，关键/紧急品项的风险估值
```

**提示词 3：模拟即将到来的需求事件库存需求**
```
请帮助计算即将到来的需求事件的库存需求和活动前订货时机。

需求事件详情：
- 事件类型：[促销 / 季节性峰值 / 生产提速 / 客户项目]
- 事件日期：[开始日期至结束日期]
- 影响SKU：[列表或描述]
- 预期需求提升：[提升X%或正常速度的X倍]
- 可比历史事件数据：[描述或粘贴，如有]

受影响SKU的正常参数：
[粘贴 SKU | 日均正常需求 | 当前库存 | 交货期 | 补货点]

对每个受影响SKU计算：
1. 活动期间总需求估算（附置信区间）
2. 活动开始时所需库存量（满足需求加安全缓冲）
3. 当前库存缺口（活动开始时所需量减去当前库存和在途订单）
4. 活动前收货的最晚安全订货日期
5. 考虑活动后需求正常化的建议订货量
6. 活动备货所需总库存投资
7. 因交货期导致活动前收货存在不确定性的风险品项
```

**提示词 4：评估供应商交货期可靠性**
```
请分析我们的供应商交货期历史记录，生成供应商可靠性评估，为安全库存和补货规划决策提供依据。

供应商数据：
[粘贴供应商表现数据——格式：供应商 | 采购订单日期 | 承诺交货日期 | 实际交货日期 | 订货数量 | 收货数量 | 是否准时（是/否）]

对每家供应商计算：
1. 平均交货期及标准差
2. 准时交付率（%）
3. 延迟时的平均延误天数
4. 最差情况交货期（第95百分位数）
5. 可靠性评级：非常可靠 / 可靠 / 不稳定 / 不可靠

然后生成：
- 从最高到最低可靠性关注度的供应商风险排名
- 按供应商可靠性等级推荐的安全库存调整系数
- 考虑观察到的变动性后当前安全库存可能不足的供应商
- 实际交货期始终短于承诺交货期的供应商——减少安全库存的机会
- 根据表现规律推荐的2-3项具体供应商改善对话
```

**提示词 5：识别并优先处理积压库存**
```
请识别我们当前库存中的积压情况并推荐减少策略。

库存数据：
[粘贴数据——格式：SKU | 描述 | 当前库存（数量）| 月均需求 | 近3个月需求 | 单位成本 | 持有成本率 | 保质期/到期日（如适用）]

积压库存定义：超过[X]周预测需求的库存

对识别出的积压品项，请提供：
1. 按积压库存价值排名的积压品项列表（超过[X]周需求的数量×单位成本）
2. 每个积压品项按当前需求速度计算的供应周数
3. 积压库存的年度持有成本
4. 风险级别：临近到期/有报废风险 / 报废风险 / 稳定（普通积压）
5. 每个品项的建议减少策略：
   - 暂停未来订单（直至库存恢复正常）
   - 加速消耗（内部替代机会）
   - 退货给供应商（如合同允许）
   - 清算/转至其他地点
   - 报废推荐（针对过时或临期库存）
6. 若实施建议措施可实现的总库存减少价值
```
