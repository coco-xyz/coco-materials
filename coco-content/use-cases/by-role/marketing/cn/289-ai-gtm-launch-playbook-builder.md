# 用例 #289：AI GTM 发布手册构建器

**角色**：市场经理 / 产品市场经理 / GTM 策略师 | **行业**：SaaS、科技、B2B 服务 | **任务**：规划、上市策略、产品发布、营收运营

---
## 详细介绍

**痛点：大多数产品发布在没有真正手册的情况下执行——结果一目了然**

上市执行差距是 SaaS 中最昂贵的战略失败之一。行业研究持续发现：50-75% 的新产品功能在发布后 90 天内未能达到其采用目标，65% 的 B2B 产品发布在第一年错过收入目标，事后分析中的主要原因不是产品质量，而是 GTM 执行：不一致的销售团队、未准备好的客户成功职能、跨渠道的不一致信息、以及没有协调计划的发布。产品发货了。市场没有注意到。责任推诿开始了。

文档缺失是根本问题。一个真正的 GTM 手册是一份 20-40 页的运营文件，它定义了：ICP（理想客户画像）的具体公司特征和行为属性、按每个买家画像校准的问题解决方案叙事、竞争定位框架、销售动作（PLG vs 销售主导 vs 渠道主导）、定价和包装逻辑、渠道策略和预算分配、带有所有者特定任务的发布时间表、按角色的赋能材料、带有定义的发布成功指标，以及发布后的反馈循环。大多数公司用 2-3 页的要点列表、一个从未被销售团队审阅的 Notion 文档，以及一个在发布当天上午 9 点发出的 Slack 公告来启动发布。

**COCO 如何解决这个问题**

COCO 的 AI GTM 发布手册构建器加速构建全面、可操作的 GTM 手册——生成完整的发布文件套件、赋能材料和协调框架，将产品发布转化为真正推动采用和收入的市场事件。

1. **ICP 和画像框架开发**：以具体性来精确定义发布面向谁以及为什么。
   - 开发具有具体公司特征标准的理想客户画像：公司规模（收入、员工数、增长阶段）、行业垂直、技术栈、组织结构和购买触发因素
   - 为购买流程中每个关键决策者和影响者构建买家画像：职位、关键职责、主要痛点、成功指标、信息来源、异议模式和偏好沟通格式
   - 绘制购买委员会：谁发起评估、谁是经济买家、谁有否决权，以及每个利益相关者需要听到什么才能推进交易
   - 识别表明潜在客户处于市场中的行为信号：触发事件、技术变化或使公司现在可能购买的组织变化

2. **定位和信息架构**：创建将在所有渠道和每个团队成员中一致使用的信息框架。
   - 定义产品的品类定位：这是在创建新品类、在现有品类内重新定位，还是替代特定的现有玩家？
   - 按画像写定位声明：这个产品对这个特定买家为什么重要，用他们描述自己问题的语言
   - 开发竞争差异化框架：如何以可辩护和有共鸣的语言针对每个主要替代方案（现有玩家、次优替代、不行动）进行定位
   - 创建信息层级结构：主要标题主张、三个支持证明点，以及每个的证据——跨网站、销售演示和营销活动使用的结构

3. **发布执行时间表和所有者分配**：构建实际使发布按时发生的运营计划。
   - 在 90 天时间表中映射所有发布活动：发布前准备（第 -8 至 -1 周）、发布周执行（第 0 天）和发布后放大（第 +1 至 +12 周）
   - 为每个活动分配具体所有者、依赖关系和完成标准——不是"市场创建发布内容"，而是"Sarah 创建博客文章（草稿截止 10 月 15 日，终稿截止 10 月 22 日，发布 10 月 28 日）"
   - 识别关键路径活动：阻断其他任务且必须按时完成否则整个时间表滑动的发布任务
   - 生成每周倒计时清单：发布前每 8 周必须完成什么，表面依赖关系

4. **销售赋能包创建**：生产销售团队从第一天起有效销售新产品所需的材料。
   - 为新产品撰写销售手册章节：发现问题、演示脚本、竞争战卡、异议处理脚本和证明点库
   - 创建简洁的"销售速查表"——销售代表在通话前可以回顾、进行正确对话的一页参考
   - 开发提案和定价指南：如何呈现定价、常见折扣情景和审批要求，以及追加销售/交叉销售触发因素
   - 为销售团队生成发布简报材料：在发布启动会上交付的幻灯片或文件，为代表提供有效销售所需的一切

5. **发布指标和反馈循环设计**：定义如何衡量成功以及团队如何在前 90 天学习和调整。
   - 定义带有具体目标的发布成功指标：采用率、试用转化、ACV 影响、推介新功能时的赢单率，以及早期采用者的 NPS
   - 创建 30/60/90 天仪表板框架：每个发布后阶段哪些指标最重要，以及它们应该触发什么行动
   - 设计客户反馈收集：询问早期采用者什么，如何构建反馈，以及如何将洞察路由回产品团队
   - 建立失败升级协议：哪些指标触发什么升级行动——何时将关注点带给收入领导层，何时独立优化

**可量化的成果**

- **发布准备度**：使用 COCO 构建手册的团队报告跨职能发布准备度评分比临时发布协调提升 40%
- **采用目标**：拥有全面 GTM 手册的产品在前 90 天达到采用目标的比率是非正式计划产品的 2.1 倍
- **销售爬坡时间**：新产品销售爬坡时间（平均代表在新产品上达成配额的时间）从 8-12 周缩短至 4-6 周
- **手册创建时间**：包含 25 页的综合 GTM 手册在 3-5 天内用 COCO 构建，相比手动开发的 3-5 周大幅缩短
- **发布一致性**：当所有团队从单一 AI 生成的信息框架工作时，跨渠道信息一致性（通过审核衡量）从 54% 提升至 87%

**受益人群**

- **产品市场经理**：快 5-10 倍地构建全面 GTM 手册——真正有时间做使定位锐利的客户研究和竞争分析，而不是将所有时间花在文件组装上
- **市场经理**：用明确的手册协调多渠道发布活动，确保每个渠道针对相同的信息和时间表执行——而不是在发布日发现销售和市场在说不同的事情
- **创始人和 CEO**：用给产品最佳成功机会的组织协调和市场准备度来发布新产品，不需要企业级 PMM 团队来构建基础设施
- **销售领导层**：以真正的赋能包进入每次产品发布——不是 PMM 团队的 3 张幻灯片概述——从第一天起就让每个代表能够进行有效的客户对话

## 实用提示词

**提示词 1：为产品发布构建完整的 GTM 手册**
```
我正在准备发布新产品（或重大功能），需要一份全面的 GTM 手册。

产品/功能详情：
- 我们在发布什么：[描述产品或功能]
- 它解决的问题：[它解决的具体客户痛点]
- 它如何工作（简短）：[用简明语言说明关键功能]
- 它与替代方案的不同之处：[什么让它比现有解决方案更好]
- 价格/包装：[定价模型、层级及其如何与现有产品配合]

发布背景：
- 目标发布日期：[日期]
- 这次发布现在重要的原因：[商业背景——竞争压力、市场时机、战略重点]
- 涉及的跨职能团队：[市场、销售、CS、产品、PR 等]
- 可用预算：[用于发布营销活动的]

目标市场：
- 主要 ICP：[理想客户是谁——尽可能具体]
- 参与购买的买家画像：[谁参与购买决策]
- 将受影响的现有客户：[应该被通知/追加销售的现有客户]

请生成：
1. 带有具体公司特征和行为标准的 ICP 定义
2. 典型购买中 2-3 个关键利益相关者的画像档案
3. 每个画像的定位声明和信息层级结构
4. 带有每周里程碑和所有者分配的 90 天发布时间表
5. 销售赋能包：销售速查表、前 5 个异议及回应、发现问题框架
6. 带有定义和 30/60/90 天目标的发布成功指标
```

**提示词 2：开发竞争定位框架**
```
我们正在进入一个竞争市场，需要清晰的定位和战卡框架。

我们的产品：[名称和描述]

我们在交易中将面对的竞争对手：
1. [竞争对手1]：[他们是什么，他们的优势，他们的典型定位]
2. [竞争对手2]：[同上]
3. [竞争对手3]：[同上]

我们的差异化优势：[我们做得更好、更不同或独特的地方]

我们相对竞争对手的弱点：[我们不够强的地方——请诚实]

我们销售的目标画像：[职位和他们最关心的事情]

常见竞争场景：
- 当我们输给[竞争对手]时：[他们说什么，我们如何输]
- 当我们赢过[竞争对手]时：[赢单模式是什么]

请：
1. 写一个清晰将我们与竞争集区分开的定位声明
2. 为每个主要竞争对手构建战卡：他们的优势、弱点、如何针对他们定位、暴露其局限性的地雷问题，以及当他们在评估中时关闭交易的证明点
3. 为销售团队创建"竞争格局"单页纸——帮助代表解释我们定位的竞争空间视觉地图
4. 写 3 个竞争对话脚本：当潜在客户提到每个竞争对手时说什么
5. 我们应该在早期发现阶段问什么"陷阱问题"来了解我们面对的是哪个竞争对手？
```

**提示词 3：撰写销售赋能发布包**
```
我们的产品下个月发布，销售团队需要一个全面的赋能包，从第一天起就能有效销售。

产品详情：
- 产品名称：[名称]
- 它做什么：[清晰、简明的描述]
- 面向谁：[主要买家和用户画像]
- 主要收益（从买家角度）：[买家实现的 3-5 个具体成果]
- 定价：[定价模型和关键价位]
- 竞争差异化：[为什么选我们而非替代方案]

销售背景：
- 管道中适用此产品的当前交易：[估计此产品影响多少现有交易]
- 销售周期长度：[从第一通电话到成交的典型天数]
- 通常如何销售：[自助 / 内部销售 / 现场销售 / 渠道]
- 预期的常见异议：[代表可能从潜在客户那里听到的]

请：
1. 写一页"销售速查表"——代表在通话前需要知道的一切
2. 创建发现问题框架：8-10 个资格确认和诊断适配度的问题
3. 为 5 个最常见预期异议写异议处理脚本
4. 开发演示大纲：赢得交易的产品演示中的 5-7 个关键时刻
5. 为评估中最常见竞争对手写竞争战卡
6. 起草给销售团队的发布启动邮件：发布什么、为什么这对他们的配额重要、以及他们本周需要做什么
```

**提示词 4：设计发布指标仪表板**
```
我需要定义我们产品发布的成功标准，并构建一个跟踪框架。

发布背景：
- 发布的产品/功能：[名称和描述]
- 发布日期：[日期]
- 商业目标：[收入目标、采用目标、战略目标]
- 目标客户细分：[我们针对的人群]

可用数据来源：[我们有权访问的分析工具和数据]

当前基准指标（发布前）：
[任何相关的发布前基准——试用率、转化率、赢单率等]

请：
1. 定义 5-7 个最重要的发布成功指标，附具体定义和目标
2. 设计 30/60/90 天仪表板：每个阶段哪些指标最重要，以及它们应该是什么样子
3. 创建警报系统：哪些指标值应该触发什么具体行动
4. 定义"发布失败阈值"——应该升级到领导层的具体指标信号
5. 写一个 90 天发布回顾模板：评估发布成功时要回答的问题和要呈现的数据
6. 我们如何在指标中区分"发布执行质量"和"产品市场契合质量"？（即，我们如何知道糟糕的发布结果是因为产品不对，还是发布执行不佳？）
```

**提示词 5：构建渠道合作伙伴 GTM 激活计划**
```
我们正在发布新产品，需要我们的渠道合作伙伴（经销商、机构、系统集成商）积极推广和销售它。

正在发布：[名称和描述]
发布日期：[日期]

渠道合作伙伴生态系统：
- 合作伙伴类型：[经销商 / 机构 / 系统集成商 / OEM 合作伙伴 / 其他]
- 活跃合作伙伴数量：[大约]
- 合作伙伴成熟度：[合作伙伴是成熟的销售者还是需要大量赋能？]
- 当前合作伙伴参与度：[今天合作伙伴的参与度如何？高/中/低]

我们需要合作伙伴做什么：
- 销售活动：[主动销售 / 与我们团队联合销售 / 仅推荐]
- 营销活动：[联合品牌活动 / 案例研究 / 活动 / 社交放大]
- 支持活动：[前线客户支持 / 实施服务]

我们可以提供给合作伙伴的：
- 财务激励：[奖励金、交易注册、利润结构]
- 营销支持：[MDF、联合品牌材料、活动支持]
- 培训和赋能：[我们可以提供什么]

请：
1. 构建合作伙伴发布激活计划：如何动员合作伙伴生态系统实现最大发布影响
2. 写合作伙伴发布简报：合作伙伴需要了解什么，他们为什么应该关心，以及这对他们有什么好处
3. 创建合作伙伴赋能包：简化战卡、联合品牌信息和他们可以分享的社交内容
4. 为发布期间（前 90 天）设计合作伙伴激励结构
5. 写一封合作伙伴招募邮件，激活当前被动的合作伙伴参与此次发布
6. 我们如何分别追踪合作伙伴对发布成功的贡献与直接销售业绩？
```
