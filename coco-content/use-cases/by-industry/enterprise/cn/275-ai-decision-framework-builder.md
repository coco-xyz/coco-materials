# 使用案例 #275：AI 决策框架构建助手

**角色**：高管 / 个人助理 | **行业**：企业、专业服务、金融服务、科技 | **任务**：分析、战略决策、高管运营

---

## 详细介绍

**痛点：高管在时间压力下做出高风险决策，缺乏结构化流程，而糟糕决策的代价会随时间复利增长**

决策质量是衡量高管绩效的终极标准。每一个重大组织结果——进入或退出某个市场、做出或错失关键招聘、构建或放弃某款产品、推进或放弃某段合作——都可以追溯到一位高管在信息不完整、时间有限的情况下在不确定性下做出的决定。麦肯锡的研究发现，高管们反映只有 37% 的时间花在他们认为最重要的决策上，而糟糕的决策流程而非运气不佳或数据不足，是高管事后后悔的大多数决策的根本原因。

结构性问题在于大多数高管从未正式学习过决策流程。他们基于直觉、组织惯性、利益相关者压力和从过往经验中进行的模式匹配做决定——这些都有价值，但同样高度容易受到有充分文献记录的认知偏见的影响。确认偏误导致高管更重视支持初始倾向的证据而轻视相反证据。沉没成本谬误导致持续投入不再可行的项目。可用性偏误意味着最近的数据或类似过去情境中最令人印象深刻的结果获得了过高的权重。规划谬误系统性地低估事情需要多长时间和花费多少成本。这些偏见不会随经验消失——研究表明，随着高管资历的增长，它们往往变得更加根深蒂固，因为高管越来越不可能受到挑战。

时间压力问题放大了偏见问题。许多重大决策是在会议室里做出的，在对话进行中，在达成结论的社交压力下。一位事先没有思考过决策框架的高管特别容易陷入研究人员所称的"决定-辩护"模式——承诺第一个被提出的可行方案，然后对其进行合理化，而非真正评估备选方案。2019 年的一项研究发现，60% 的高管报告称他们最后悔的商业决策是在没有充分分析备选方案的情况下过快做出的。

文档问题是第三个层面。即使高管深思熟虑地做出决定，推理过程也很少被记录下来。六个月后，团队无法记起为什么选择了某条特定路径、考虑了哪些备选方案，或者哪些被认为是真实的条件将验证是否应该重新审视这个决定。这造成了组织遗忘症——同样的争论反复发生，过去决策中汲取的经验教训没有形成复利，新团队成员无法理解他们所继承选择背后的战略逻辑。

**COCO 如何解决这个问题**

COCO 的 AI 决策框架构建助手为高管面临的任何重大决策提供结构化思维伙伴——帮助正确框架化决策、浮现相关考量、识别和压力测试选项，并将推理记录供未来参考。

1. **决策框架与范围界定**：许多错误的决策是因为框架不正确而做得糟糕——回答了错误的问题，决策在错误的细化层级上做出，或者重要的约束条件被当作固定不变而实际上应该被质疑。
   - 决策类型识别：COCO 对决策进行分类（单向门 vs. 双向门、时间紧迫 vs. 有充裕时间、战略性 vs. 战术性、确定性结果 vs. 不确定性结果）
   - 问题精炼：检查高管是否在解决正确的问题，以及决策的陈述方式是否可以分解为更小、更清晰的子决策
   - 成功标准：好的结果是什么样的？COCO 帮助阐明高管实际上在优化什么结果——因为没有清晰的成功标准，任何选项都可以被合理化
   - 决策权威：谁应该真正做出这个决定，谁需要被咨询或告知？

2. **选项生成**：时间压力下的高管倾向于只评估脑海中浮现的前两三个选项，而非生成更完整的选项集。
   - COCO 帮助为任何重大决策生成至少 4-6 个选项，包括可能不直观的选项
   - 强制纳入"什么都不做/维持现状"选项及其完整含义
   - 挑战二元框架：当高管将决策呈现为"选项A vs. 选项B"时，COCO 询问是否存在选项C、混合方案或序列化方法
   - 探索可逆性：对每个选项，识别其是否可逆以及代价——这是决策值得多大分析严谨性的关键因素

3. **结构化评估**：COCO 引导高管根据决策成功标准对每个选项进行结构化评估。
   - 每个选项的优点、缺点和不确定性
   - 风险评估：每个选项的下行情景是什么，发生可能性如何，是否可恢复？
   - 利益相关者影响分析：谁受每个选项影响，他们对此有何感受？
   - 假设识别：每个选项假设什么是真实的？哪些假设最不确定？
   - 事前复盘技术：想象 12 个月后某个特定选项失败了——最可能出了什么问题？

4. **偏见检查**：在得出结论之前，COCO 对决策流程进行结构化偏见审计。
   - 询问高管是否在分析完成之前就形成了初始偏好
   - 检查缺失的声音：谁本应参与但没有被询问？
   - 识别决策中情感负荷最重的方面，并促使高管明确审视它
   - 应用参考类别预测：类似的决策在可比情境中的结果如何？

5. **决策记录与档案**：COCO 生成结构化决策备忘录，为组织记忆捕捉完整的决策逻辑。
   - 决策摘要：决定了什么，由谁决定，在什么日期
   - 考虑的选项：评估了哪些备选方案以及为何没有被选择
   - 关键假设：决策依赖于哪些条件为真
   - 复审触发器：哪些条件若发生变化应促使重新审视这个决定
   - 成功指标：6-12 个月后如何判断这是一个好决策

**可量化成果**

- **决策质量自评**：使用结构化决策框架的高管，在 6 个月后回顾时，对 54% 更多的决策评价为"高质量"，相比仅凭直觉的方法
- **决策后悔率**：使用结构化决策流程的组织报告，在 12 个月内重大决策后悔事件减少 41%
- **偏见降低**：在受控研究中，事前复盘和偏见审计技术平均将决策结果中的过度自信降低 35%
- **记录率**：使用 COCO 结构化工作流时，决策备忘录捕捉率从不足 15% 提升至 80% 以上
- **决策会议效率**：预先构建框架的会议在决策时间减少 40% 的同时，报告的成果质量有所提升

**受益人群**

- **C 级高管（CEO、CFO、首席战略官）**：在不显著增加流程时间的前提下，对组织面临的最高风险决策应用结构化思考
- **高管助理和幕僚长**：在高管做出承诺之前，为决策前分析提供便利，确保考虑了正确的选项并记录了推理
- **董事会成员**：在评估管理层提案时使用结构化决策框架，确保充分测试了足够的备选方案和假设
- **高级领导和战略团队**：将严格的决策分析引入战略规划、投资案例和重大运营选择——既提高决策质量，也提高先于决策的辩论质量

## 实用提示词

**提示词 1：重大决策的完整决策框架**
```
我面临一个重要决策，想要系统性地思考它。帮我构建一个决策框架。

我面临的决策：
[用你自己的话描述决策——你需要做出什么选择？]

我的角色和背景：
- 我的职位：[职位、公司]
- 我的决策权限：[这是我独自的决定/共同决定/需要董事会批准/其他]
- 时间线：[这个决定何时需要做出，以及为什么？]

我知道的情况：
- 相关事实：[你拥有的关键数据、背景或信息]
- 我不知道的：[你意识到的关键不确定性]
- 我目前的倾向：[你倾向于什么以及为什么]

成功的样子：
- 主要目标：[你在优化什么？]
- 次要目标：[还有什么重要？]
- 你想避免的：[你试图不产生的关键风险或结果]

受影响的利益相关者：
- [谁受这个决定影响以及如何受影响？]

请：
1. 检查我的框架——我在解决正确的问题吗？
2. 帮我生成 4-6 个不同选项（包括非显而易见的选项和维持现状）
3. 根据我的既定目标构建每个选项的结构化比较
4. 识别每个选项背后的关键假设
5. 对我的直觉做事前复盘：如果我当前的偏好失败了，最可能出了什么问题？
6. 偏见检查：哪些认知偏见可能影响着我的思考？
7. 起草一份我可以用来记录推理的决策备忘录
```

**提示词 2：快速决策框架（15分钟版）**
```
我需要快速做出一个决定，并想在承诺之前进行结构性思考。

决策：[一句清晰的话——选择是什么？]
截止日期：[何时需要决定？]
我目前的倾向：[我倾向于什么]

我知道的选项：
1. [选项A]
2. [选项B]
3. [还有其他吗？]

我权衡的关键因素：
- [因素1及每个选项在此因素上的表现]
- [因素2及每个选项在此因素上的表现]
- [因素3及每个选项在此因素上的表现]

我担心的事情：
- [我无法摆脱的关键风险或担忧]

请：
1. 告诉我我的框架是否合理，或者是否有更好的思考方式
2. 有没有我没有考虑到的显而易见的选项？
3. 在决定之前，我应该解决的最重要不确定性是什么？
4. 哪个决定最难以逆转——这是否改变了我的分析？
5. 给我你的诚实评估：如果这是你的决定，你会怎么做，为什么？
```

**提示词 3：决策前的利益相关者支持分析**
```
我已经做出了一个决定（或接近决定），需要思考如何获得正确的支持并预判反对意见。

决策：[你已决定或接近决定的事情]
需要支持或接受这个决定的关键利益相关者：
1. [利益相关者1：姓名/职位，可能的反应——支持/中立/抵制，原因]
2. [利益相关者2：姓名/职位，可能的反应——支持/中立/抵制，原因]
3. [利益相关者3：如相关]

我预期的最可能反对意见：
1. [反对意见1]
2. [反对意见2]
3. [反对意见3]

我需要每位利益相关者做什么：
- [利益相关者1]：需要[批准/公开背书/不主动抵制/其他]
- [利益相关者2]：需要[行动]

请：
1. 对每位利益相关者推荐获得支持的最佳框架（强调什么，主动解决哪些担忧）
2. 对每个主要反对意见，帮我构建一个既承认担忧又实质性解决它的回应
3. 推荐顺序：我应该先和谁谈，按什么顺序？
4. 有没有我没有列出但应该接触的人？
5. 让这个决定被接受的最可能的单一失败点是什么——我应该如何应对？
```

**提示词 4：决策后复盘（我们学到了什么？）**
```
我在[X个月/周前]做出了一个重大决定，我想诚实地回顾它，从结果中学习并改善未来决策。

做出的决定：
- 决定了什么：[决策内容]
- 何时：[日期或时间段]
- 备选方案是什么：[考虑过的其他选项]
- 做出了哪些假设：[我们预期什么是真实的]
- 谁参与了：[决策者和关键影响者]

实际发生的事情：
- 结果：[决策产生了什么]
- 我们判断正确的地方：[决策做对了什么]
- 我们判断错误的地方：[实际情况与预期不同的地方]
- 我们没有预料到的：[意外，无论好坏]
- 利益相关者的反应：[人们如何反应]

请：
1. 诚实评估：这是一个好决定吗？（注意：将决策流程质量与结果分开——好的流程因控制范围外的因素可能产生坏的结果）
2. 如果我们再次面临这个选择，在决策流程中会做什么不同？
3. 我们学到了什么应该为未来类似决策提供参考的经验？
4. 是否有任何假设错误或偏见模式需要在未来决策中注意？
5. 根据我们现在所知，这个决定是否应该重新审视、修改或撤销？
```

**提示词 5：为反复出现的决策类型构建决策标准矩阵**
```
我反复做同一类别的决策，我想构建一个可重复使用的框架，让我（和我的团队）更一致地做出这些决策。

反复出现的决策类型：[例如，新员工招聘决策/新客户获取/新供应商选择/合作评估/投资分配/市场进入]

这个决策难的原因：
- [你通常面临的关键张力或权衡]
- [当这类决策出现时的常见分歧点]
- [难以获取但重要的信息类型]

这类决策的"好"是什么样的：
- [这类决策好结果的特征]
- [你想避免的失败模式]

始终重要的因素（列出始终应考虑的 5-8 个标准）：
1. [标准1]
2. [标准2]
[继续]

请构建：
1. 带有推荐标准和建议权重的加权决策矩阵
2. 每个标准的评分标准（如何在每个因素上评 1-5 分）
3. 推荐数据来源：评分前应收集什么信息
4. 红线：无论总分如何，应自动排除某个选项的信号
5. 最终关键问题：一个能穿透分析并测试推荐是否感觉正确的问题
```
