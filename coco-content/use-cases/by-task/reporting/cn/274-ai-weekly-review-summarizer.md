# 使用案例 #274：AI 每周回顾总结助手

**角色**：高管 / 个人助理 | **行业**：企业、专业服务、金融服务、科技 | **任务**：报告、高管运营、绩效监控

---

## 详细介绍

**痛点：高管在每周结束时对"真正做了什么、哪里有风险、下周需要什么"毫无清晰认知**

每周回顾是高管效率领域最被普遍认可的实践之一，也是最普遍被忽视的实践之一。概念简单：在一周结束之前，从运营噪音中抽身，回顾发生了什么，评估哪些在推进、哪些没有，并为下周设定清晰的优先事项。哈佛商学院的研究发现，进行结构化每周回顾的高管实现季度优先目标的概率高出 28%，在日常决策中也报告显著更高的清晰度和信心。

这一实践在执行层面失败的原因不是缺乏意愿，而是认真做好它的成本太高。一次全面的每周回顾需要从多个不相连的来源汇聚信息：关键对话成果的邮件线索、会议次数分布的日历记录、项目管理工具中的里程碑状态、CRM 中的商机进展、财务仪表板上的关键指标，以及高管自己的笔记和记忆——而这些背景信息根本没有存储在任何结构化的地方。将这一切综合成一幅连贯的画面——什么在推进、什么停滞、什么有风险、做了哪些决定——需要 60 至 90 分钟的专注工作。在繁忙的高管日程中，这是一块与所有其他事项竞争的宝贵时间。

通常发生的替代行为是三种不够充分的方式之一。部分高管进行"心理回顾"——快速、非结构化地回想这一周，只捕捉到最显眼的事件，错失了需要数据分析才能看到的模式级洞察。另一些人依赖团队的状态更新——汇编各部门自我汇报的成就摘要，这些内容经过高度过滤，倾向于淡化问题，也无法给高管提供关于自身时间和注意力分配的个人视角。第三类人干脆不回顾——他们在周一凭借上周的惯性启动，对收件箱中最先出现的事项做出反应，没有对应该优先处理的事项进行深思熟虑的判断。

每周回顾不足的复利成本是战略漂移：高管时间分配与组织真正需要关注的领域之间逐渐扩大的差距。数月后，这表现为：某些优先事项获得了一周的关注便慢慢淡出视野，关系未能以合适的频率维护，以及总有一种感觉——每周结束，真正重要的事情上的进展几乎可以忽略不计。

**COCO 如何解决这个问题**

COCO 的 AI 每周回顾总结助手将每周回顾从 90 分钟的手动综合任务转变为 20 分钟的结构化反思，产出完整的一周画面——完成了什么、哪里未达预期、时间花在了哪里、做了哪些决定、下周需要关注什么。

1. **每周活动综合**：COCO 汇聚本周关键输入，按高管关心的维度生成结构化摘要。
   - 会议情报：本周发生了哪些重要会议，按类型分类（战略型、运营型、对外型、内部型），标注关键成果
   - 沟通快照：已解决或仍待处理的高量或高重要性邮件线索，已发送的重要外部沟通
   - 决策日志：本周做了哪些实质性决定，由谁做出，记录的决策依据
   - 承诺追踪：本周高管对他人做出了哪些承诺，他人应履行的承诺状态如何

2. **优先级进展评估**：COCO 将本周实际活动与高管在周初设定的优先事项进行对比。
   - 针对每项既定优先事项：为其做了哪些工作，成果如何，是推进了还是停滞了，以及原因
   - 精力分配：高管时间用于核心优先事项 vs. 运营/被动处理活动 vs. 未计划需求的粗略比例
   - 停滞项识别：出现在上周计划中但本周没有实质推进的优先事项——并分析原因和解锁路径

3. **风险与问题浮现**：识别可能被遗漏或若不处理将成为问题的事项。
   - 临近截止日期但尚未完成的承诺
   - 在有意义的时间段内未保持联系的关系
   - 被推迟但需要做出的决定
   - 本周浮现的规律——反复出现的主题、持续的摩擦点、早期预警信号

4. **下周框架**：COCO 帮助高管以主动设定的优先事项而非被动惯性开启下周。
   - 基于本周评估推导出的下周前 3 大重点优先事项
   - 下周日历中已有的、需要准备的会议和承诺
   - 基于关系维护逻辑主动触达的对象
   - 基于本周规律，建议考虑的一项结构性改变

5. **纵向趋势追踪**：跨多周，COCO 维护动态记录，支持规律层面的回顾。
   - 月度汇总：哪些优先事项保持稳定进展，哪些反复停滞
   - 关系维护频率：谁获得了定期关注，谁被忽视了
   - 决策速度：高管做决定是否足够快，是否在制造瓶颈
   - 会议 ROI：哪些定期会议在产出成果，哪些在消耗时间而没有明确回报

**可量化成果**

- **每周回顾完成率**：使用 COCO 的高管完成结构化每周回顾的一致性是手动方法的 4 倍（平均完成率 87% vs. 手动 22%）
- **优先事项达成率**：结构化每周回顾与季度优先事项达成率提升 28-35% 呈正相关
- **战略漂移降低**：高管在进行 8 周结构化每周回顾后，报告既定优先事项与实际时间分配之间的偏差减少 45%
- **停滞项检测**：每周未被发现的平均停滞优先事项从 2.1 项降至 0.3 项遗漏
- **时间投入**：使用 COCO 完成完整每周回顾需 18-25 分钟，而手动方式需 60-90 分钟，或根本没有进行

**受益人群**

- **C 级高管（CEO、COO、总裁）**：无需投入 90 分钟周五下午时间进行手动综合，即可保持战略清晰度和对季度优先事项的个人问责
- **高管助理和幕僚长**：为高管生成涵盖所有活动流的结构化每周简报——这项任务此前需要 2-3 小时的调研和整合
- **高级领导和副总裁**：使用一致的框架与直接汇报团队开展有纪律的每周回顾，追踪承诺、进展和阻碍
- **创始人和创业者**：无需大型组织的基础设施，即可保持战略纪律，在季度漂移积累之前尽早发现

## 实用提示词

**提示词 1：完整每周回顾**
```
帮我做每周回顾。我会分享本周发生的事情，我想要一份结构化评估和下周的计划。

我的角色：[职位、公司、团队规模]
本季度我的前3大优先事项：[列出]
本周我为自己设定的具体优先事项：[你计划专注的内容]

本周发生的事情：
- 重要会议（列出重要的）：[会议名称/目的——成果或状态]
- 主要成就或成果：[真正完成并感到满意的事情]
- 没有发生或停滞的事情：[计划了但没有推进的，或遇到阻碍的]
- 做出的重要决策：[决定了什么，由谁决定]
- 我对他人做出的承诺：[谁、什么事、何时]
- 他人欠我的承诺：[谁、什么事、状态]
- 出现的问题或担忧：[任何让你担心的新情况]
- 时间观察：[对本周时间分配的粗略感知——什么事情意外消耗了时间]

请提供：
1. 本周摘要：本周发生的 3-5 件最重要的事情是什么？
2. 优先事项进展：我的每项既定优先事项本周有何进展？
3. 值得认可的成果：哪些进展应该被承认为真正的进步？
4. 担忧和风险：在成为问题之前，哪些事项需要我的关注？
5. 推荐的下周优先事项前三——附推理
6. 一项结构性观察：本周是否有规律表明我应该改变工作方式？
```

**提示词 2：快速每周快照（15分钟版）**
```
我有15分钟做每周回顾。帮我快速获取最重要的洞察。

本周快速快照：
- 最好的事情：[一句话]
- 最大的挫折或阻碍：[一句话]
- 最重要的会议成果：[一句话]
- 应该做但没做的事：[一句话]
- 意外的时间黑洞：[一句话]

进入本周时我设定的优先事项是：
1. [优先事项1] — 进展：[向前推进/停滞/已完成]
2. [优先事项2] — 进展：[向前推进/停滞/已完成]
3. [优先事项3] — 进展：[向前推进/停滞/已完成]

下周安排：[列出已排定的主要会议或承诺]

请给我：
1. 一段话的本周摘要
2. 每项优先事项的红绿灯评估（绿/黄/红），附一行背景说明
3. 下周三件不能让它们被遗漏的事
4. 按重要性排序的下周前3大优先事项
```

**提示词 3：月度回顾与季度进展评估**
```
我想在本月底做一次更深入的回顾。帮我评估季度目标进展，并规划最后阶段。

我的季度优先事项（列出，如有则附上目标指标）：
1. [优先事项1]：[目标成果或指标]
2. [优先事项2]：[目标成果或指标]
3. [优先事项3]：[目标成果或指标]

当前在季度中的位置：[第X周，共Y周]
季度剩余天数：[X天]

本季度逐月进展：
- 第1个月：[发生了什么——主要里程碑、成就、挫折]
- 第2个月：[发生了什么]
- 本月（第3个月）：[到目前为止发生了什么]

每项优先事项的当前状态：
1. [优先事项1]：[诚实的当前状态——按计划/落后/有风险/已完成]
2. [优先事项2]：[诚实的当前状态]
3. [优先事项3]：[诚实的当前状态]

请：
1. 基于当前轨迹评估每项季度目标的达成概率
2. 识别在剩余时间内集中投入能产生最高影响的一两个优先事项
3. 标注任何已经结构性停滞、可能需要修订或推迟的优先事项
4. 推荐如何在剩余[X]天内分配我的专注力
5. 起草一份可分享给团队或董事会的月度摘要
```

**提示词 4：每周团队问责回顾**
```
我需要回顾本周团队进展，并为周一团队会议做准备。

我的团队结构：[列出直接汇报成员及其职能领域]

我掌握的每位团队成员状态更新（粘贴或概述你所知道的）：
- [团队成员1]：[完成了什么、进行中的、任何阻碍]
- [团队成员2]：[完成了什么、进行中的、任何阻碍]
[继续列出所有成员]

上周团队做出的关键承诺：
1. [谁承诺了什么，截止何时——以及状态]
2. [继续]

我们本季度的共同团队优先事项：
1. [优先事项1]
2. [优先事项2]
3. [优先事项3]

请：
1. 用5个要点总结所有职能的团队进展
2. 识别逾期或有滑坡风险的承诺
3. 标注工作量或状态值得直接对话的团队成员
4. 推荐周一团队会议的前3个议程项——附每项的正确框架
5. 本周团队整体的情况是否有什么应该改变我下周的优先事项或关注点？
```

**提示词 5：年度回顾与年度目标评估**
```
我想做一次全面的年度回顾，并以此规划下一年。帮我诚实评估这一年，并识别我想做出什么改变。

这一年的概况：
- 我的组织/角色：[职位、公司、团队]
- 我主导或负责的主要工作：[列出今年最重要的 4-6 件工作]
- 今年的主要成就：[你最引以为豪的事情]
- 主要失望或未达标：[哪里未达到你的预期]
- 最大的意外（正面和负面）：[你没有预料到的事情]
- 消耗时间超过应有水平的事：[注意力分配不当的地方]
- 应该但没有得到足够关注的事：[你希望投入更多的事]

今年的人际关系：
- 我巩固的关系：[谁，如何巩固的]
- 我忽视的关系：[谁，发生了什么]
- 我建立的新关系：[他们是谁，为什么重要]

技能与成长：
- 今年我学到的：[新能力、知识、视角]
- 我计划但没有推进的：[打算发展但未付诸行动的技能或领域]

请提供：
1. 年度摘要：定义这一年对我最有影响的 5-7 件事
2. 诚实评估：我在哪里增值最多，在哪里效果最差
3. 规律识别：有哪些反复出现的主题（正面和负面）需要我关注？
4. 针对下一年：我应该开始做什么、停止做什么、继续做什么？
5. 推荐的下一年前5大优先事项，附各项推理
```
